# BBS BATTLE CHAOS

匿名掲示板に「投資」を足したらどうなるか、を試してる個人開発のゲームです。

普通の掲示板みたいに誰でも匿名でスレを立てて書き込めます。違うのは、登録した人が「これは面白い」と思ったレスにゲーム内のお金を投資して、株を買えるところ。後から投資が集まったレスほど株価が上がるので、**まだ誰も気づいてない名レスを早めに仕込んで、値上がりしたら売って利確する**——要するに目利き勝負＋売買のタイミング勝負です。

あと、スレもレスも放っておくと時間で朽ちて消えます。盛り上がってるうちに動かないと、買った株も売り抜けられず紙くずになる。そういう焦りも込みで遊ぶやつです。

世界には「シーズン」があって、一定期間（既定1週間）が経つと1シーズンが終了。最終資産でランキングが確定して、自動的に次のシーズンが始まります。残り時間はヘッダーのカウントダウンで確認できます。

> 念のため：ゲーム内のお金や株に現実の価値はありません。換金もできないし、賭博でもないです。

## ざっくりルール

- スレ立て・書き込みは匿名・無料。ログインも要りません。
- 売買（株の購入・売却）は要ログイン。初期資金が配られます。
- 株価は累計投資額で上がる（早く買うほど安い＝ボンディングカーブ）。
- **株はいつでも売れる**。受け取り額は「いま売ったら得られる額」＝集まった投資の持ち分 × 鮮度(HP)。朽ちるほど目減りするので、売り抜けが肝。
- レスもスレもHPがあって、時間とともに減る。0になると消滅（dead）。dead の株はほぼ0。
- 投資が集まったレスはレベルが上がって寿命が延びる。
- 相場には「天候」があって、荒れてると朽ちるのが速い。
- 人が少ないうちは運営のNPCが投資して場を動かす（ランキングには入らない）。
- 通貨総量には天井があり（NPCの資金供給は上限で止まる）、インフレ/デフレが一方向に振り切れないようにしてあります。
- **シーズン制**：一定期間（既定1週間）ごとにシーズンが締まり、総資産ランキングを確定 → 自動リセットして次シーズンへ（所持金・スレ・株は初期化）。残り時間はヘッダーに表示されます。

## 技術構成

- PHP 8 / MySQL 8 / nginx、全部 Docker
- フレームワークは使わず、ルーター・ビュー・DIあたりは自前。DIコンテナだけ PHP-DI
- クリーンアーキテクチャ寄りの構成（Domain / Application / Infrastructure / Presentation）
- テストは PHPUnit（ユニット中心 + DB結合テスト）
- 管理画面（モデレーション・通報対応・BAN・お問い合わせ・ラウンド管理・ゲームバランス編集・監査ログ）あり

なんで自作かというと、勉強がてら中身を全部見えるようにしたかったからです。
Claude Codeで作ってもらいつつ、脆弱性は自分がチェックしている感じです。

## ローカルで動かす

Docker があれば動きます。

```bash
cp .env.example .env   # 中身を環境に合わせて埋める
docker compose up -d
```

`migrate` サービスがDBのスキーマを流してから `php` が立ち上がります。起動したらブラウザで:

```
https://localhost:18443
```

開発用の証明書は自己署名なので、ブラウザに警告が出ます。承認して進んでください。HTTP(18723)はHTTPSへ飛ばしてます。

メールはローカルだと Mailpit で受け取れます（確認メール・パスワード再設定など）。UIは `http://localhost:18025`。

## よく使うコマンド

```bash
# テスト
composer test
# または
docker compose exec php vendor/bin/phpunit

# マイグレーション（手動で流したいとき）
docker compose exec php php bin/migrate.php

# ゲーム進行を1ティック手動で回す（NPC・掃除・終局なら自動リセット）
docker compose exec php php bin/cron.php

# ラウンドの強制リセット（終局していなくても仕切り直す）
docker compose exec php php bin/round.php --force

# 管理者に昇格（メールアドレス指定）
docker compose exec php php bin/admin.php promote you@example.com
```

## ゲームの進行（cron）

NPCの活動・期限切れ掃除・終局時の自動リセットは `GameTick` がまとめて担当します。駆動方式は2つ:

- **web（既定 / cron不要）**：公開GETに相乗りして進む。開発や小規模ならこれでOK。
- **cron（本番推奨）**：`.env` で `GAME_TICK_DRIVER=cron` にし、`* * * * * php bin/cron.php` を回す。トラフィックに依存せず確実に進む。

詳細は [docs/運用_ラウンドとログ.md](docs/運用_ラウンドとログ.md) を参照。

## 設定（.env）

`.env.example` にひととおりコメント付きで書いてあります。要点だけ:

- `DB_PASSWORD` などは本番だと弱いと起動を蹴られます。
- `APP_SECRET`（16文字以上）は本番必須。投稿者のIP匿名ハッシュ(HMAC)に使います。変更すると過去の author_hash / IP BAN は無効化されます。
- `TRUSTED_PROXIES`：本番のリバースプロキシ(Caddy等)配下で実クライアントIPを取るために設定（未設定だと全員プロキシIP扱いになり、IP BAN・連投制限が機能しません）。
- `GAME_TICK_DRIVER`：`web`（既定）か `cron`。本番は cron 推奨。
- メールは本番だと実送信(SMTP)必須。
- Googleログイン、ゲームバランス、法務ページの運営者名などは環境変数で切り替え。ゲームバランスは管理画面からも編集できます。

## ディレクトリ

```
src/
  Domain/          ルール・エンティティ（外部に依存しない核）
  Application/     ユースケース・ポート・サービス（GameTick / MarketSimulator など）
  Infrastructure/  DB・メール・OAuth・ログ・セキュリティ(IPハッシュ)などの実装
  Presentation/    ルーティング・コントローラ・ビュー・多言語・管理画面
bin/               migrate / cron（進行） / round（強制リセット） / admin（昇格）の CLI
database/migrations/  連番SQL
tests/             ユニット中心。結合テストはDB必須（普段はスキップ）
docs/              設計と運用メモ
```

日本語・英語の両対応です（ヘッダーから切り替え）
