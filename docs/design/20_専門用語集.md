## A. Webの基本

### リクエスト / レスポンス
- **ざっくり**：ブラウザからの「お願い」がリクエスト、サーバーからの「返事」がレスポンス。
- **詳しく**：ブラウザがURLを開く＝サーバーにリクエストを送る。サーバーは処理して、HTMLやJSONなどをレスポンスとして返す。
- **このプロジェクト**：お願いを表す箱が [`Request`](../../src/Presentation/Http/Request.php)、返事を表す箱が [`Response`](../../src/Presentation/Http/Response.php)。

### HTTPメソッド（GET / POST など）
- **ざっくり**：リクエストの「種類」。**GET＝見る／取得**、**POST＝送る／作成・更新**。
- **詳しく**：同じURLでもメソッドが違えば別の操作。`GET /thread/5`（5番を見る）と `POST /thread/5`（5番に何か送る）は別物。
- **このプロジェクト**：`$router->get(...)` と `$router->post(...)` で使い分ける。メソッドが違うと **405** になる。

### URL / パス / クエリ
- **ざっくり**：`https://example.com/thread/5?sort=new` のうち、
  - **パス** = `/thread/5`（道順の本体）
  - **クエリ** = `?sort=new`（おまけの条件）
- **このプロジェクト**：パスは `$request->path()`、クエリは `$request->query('sort')` で取れる。ルーティングが見るのは**パスとメソッド**。

### ステータスコード（200 / 404 / 405 / 500 など）
- **ざっくり**：返事につく「結果を表す番号」。
- **詳しく**：
  - **200** OK（成功） / **201** Created（作成成功）
  - **302** リダイレクト（別URLへ移動して、の意）
  - **403** Forbidden（禁止。検問で弾かれた）
  - **404** Not Found（そんなURL無い）
  - **405** Method Not Allowed（URLはあるけどメソッド違い）
  - **500** Internal Server Error（サーバー内部でエラー）
- **このプロジェクト**：[index.php](../../public/index.php#L20) が例外を見て 404 / 405 / 500 に振り分ける。

### ヘッダー
- **ざっくり**：本文とは別に付ける「付箋情報」。
- **詳しく**：返事の種類（`Content-Type`）や、移動先（`Location`）、許可メソッド（`Allow`）などを伝える。
- **このプロジェクト**：`Response::withHeader('Allow', 'POST')` のように付ける。

### HTML / JSON
- **ざっくり**：**HTML** = 人が見る画面の文書。**JSON** = プログラム同士でやり取りするデータ形式。
- **このプロジェクト**：`Response::html(...)`（画面用）と `Response::json(...)`（API用）を使い分ける。

## B. ルーティング用語

### ルーティング / ルーター
- **ざっくり**：「このURLのときはこの処理」と**振り分けること**がルーティング。やる係が**ルーター**。
- **このプロジェクト**：[`Router.php`](../../src/Presentation/Routing/Router.php) が担当。電話の交換手のイメージ。

### ルート
- **ざっくり**：「URL ＋ メソッド → 担当」の**1本の対応づけ**。
- **このプロジェクト**：1本が [`Route`](../../src/Presentation/Routing/Route.php) オブジェクト。一覧は [`routes.php`](../../src/Presentation/Routing/routes.php)。

### 静的ルート / 動的ルート
- **静的ルート**：URLが完全固定。例 `/thread/create`。一瞬で一致判定できる（速い）。
- **動的ルート**：URLの一部が変わる。例 `/thread/{id}`。`/thread/5` でも `/thread/99` でも一致する。
- **このプロジェクト**：Router は**静的を先に**、無ければ**動的を上から順に**調べる（速い方を先に）。

### パスパラメータ（`{id}`）
- **ざっくり**：URLの中の**変数部分**。`{ }` で囲んで書く。
- **詳しく**：`/thread/{id}` の `{id}` は、`/thread/5` なら `id = 5` として中身を取り出せる。
- **このプロジェクト**：コントローラーで `$request->param('id')` として受け取る。

### resolve（解決） / dispatch（実行）
- **resolve**：「このURLはどのルート？」を**探す**段階。見つからなければ 404/405。
- **dispatch**：見つけたルートを使って、ミドルウェア → コントローラーと**実際に動かす**段階。
- **このプロジェクト**：[`Router::resolve()`](../../src/Presentation/Routing/Router.php#L59) と [`Router::dispatch()`](../../src/Presentation/Routing/Router.php#L99)。

### RouteMatch
- **ざっくり**：「**見つかったルート ＋ 抜き出した値**」をまとめた案内メモ。
- **このプロジェクト**：[`RouteMatch`](../../src/Presentation/Routing/RouteMatch.php)。`route`（どのルートか）と `params`（`['id' => '5']` など）を持つ運搬用の入れ物。

## C. しくみ・部品の名前

### フロントコントローラー
- **ざっくり**：**全リクエストの唯一の入口**になるファイル。
- **詳しく**：どんなURLでもまずここを通す設計。共通の準備（セッション・ルーター起動など）を1箇所にまとめられる。
- **このプロジェクト**：[`public/index.php`](../../public/index.php)。

### コントローラー
- **ざっくり**：URLごとの**担当者**。入力を受け取り、返事（Response）を作る。
- **詳しく**：複雑な業務ロジックそのものは持たず、「入力を取り出す → 処理を呼ぶ → 返事を作る」の司令塔に徹する。
- **このプロジェクト**：[`Controller/`](../../src/Presentation/Controller) 配下。`[ThreadController::class, 'show']` のように「クラスとメソッド名」で指定される。

### ハンドラー
- **ざっくり**：「このルートで**呼ぶ処理**」の指定。ここでは `[クラス, 'メソッド名']` の組。
- **このプロジェクト**：`Route` の `handler` プロパティ。`dispatch()` が `new クラス()` して `メソッド()` を呼ぶ。

### ミドルウェア
- **ざっくり**：コントローラーの**前後にはさむ共通の検問処理**。
- **詳しく**：通すなら「次へ（`$next`）」を呼ぶ、止めるなら自分で `Response` を返す。複数あると玉ねぎ状に重なる。
- **このプロジェクト**：[`Middleware/`](../../src/Presentation/Routing/Middleware) 配下。今は CSRF チェックの [`CsrfMiddleware`](../../src/Presentation/Routing/Middleware/CsrfMiddleware.php) がある。

### ミドルウェアパイプライン / `$next`
- **ざっくり**：ミドルウェアを**一列に連結した通り道**。`$next` は「次の処理」を指す合言葉。
- **詳しく**：各ミドルウェアは `$next($request)` を呼ぶことで次にバトンを渡す。最後のバトン先がコントローラー本体。
- **このプロジェクト**：[`dispatch()` の `array_reduce`](../../src/Presentation/Routing/Router.php#L113) が、登録の逆順に内側から包んでパイプラインを組み立てる。

### MiddlewareFactory（ファクトリ）
- **ざっくり**：名前（`'csrf'`）から**実物のミドルウェアを作る工場**。
- **詳しく**：「ファクトリ＝必要なものを名前で注文すると作って渡してくれる係」。ルート側は名前を書くだけでよい。
- **このプロジェクト**：[`MiddlewareFactory::make('csrf')`](../../src/Presentation/Routing/MiddlewareFactory.php)。`'rate_limit:post'` のように `:` で引数も渡せる。

### 例外（Exception）/ throw / catch
- **ざっくり**：処理の途中で起きた「異常」を表す合図。**throw**＝合図を投げる、**catch**＝受け止める。
- **詳しく**：異常時に普通の戻り値で返す代わりに例外を投げ、呼び出し元でまとめて受け止めて対応できる。
- **このプロジェクト**：ルートが無ければ [`NotFoundException`](../../src/Presentation/Routing/NotFoundException.php)、メソッド違いは [`MethodNotAllowedException`](../../src/Presentation/Routing/MethodNotAllowedException.php) を throw。index.php が catch して 404/405/500 に変換。

## D. PHP・実装まわり

### 名前空間（namespace） / `use`
- **ざっくり**：クラスの**住所**。同じ名前のクラスがぶつからないように区切る仕組み。
- **詳しく**：`App\Presentation\Routing\Router` のような階層。`use` は「このファイルでは短い名前で呼ぶよ」という宣言。
- **このプロジェクト**：`src/` 以下のフォルダ構成と住所（`App\...`）が対応している。

### オートロード（autoload）
- **ざっくり**：使うクラスを**自動で読み込んでくれる**仕組み。`require` を毎回書かなくてよい。
- **このプロジェクト**：[index.php](../../public/index.php#L5) 冒頭の `require '.../vendor/autoload.php'`（Composer製）。

### セッション
- **ざっくり**：ブラウザごとに**サーバー側で覚えておく記憶**。ログイン状態やCSRFトークンの保管に使う。
- **このプロジェクト**：`session_start()` で開始。`$_SESSION['_csrf']` などに入る。

### 連想配列（map） / `$_GET` `$_POST` `$_SERVER`
- **ざっくり**：**キーと値の対応表**。`['id' => '5']` のように名前で値を引ける配列。
- **詳しく**：`$_GET`（クエリ）・`$_POST`（フォーム送信）・`$_SERVER`（サーバー情報）は、PHPが用意する“生の”連想配列。
- **このプロジェクト**：生のまま触らず、[`Request::fromGlobals()`](../../src/Presentation/Http/Request.php#L18) で `Request` 箱にまとめてから使う。

### 正規表現（regex）
- **ざっくり**：「**文字列のパターン**」を表す書き方。「数字だけ」「英字3文字」などの形を1行で表せる。
- **詳しく**：このプロジェクトで出てくる記号：
  - `^ ... $` … 「先頭から末尾までピッタリ」
  - `\d` … 数字1文字（`\d+` で「1文字以上の数字」）
  - `+` … 「直前のものが1回以上」
  - `[^/]+` … 「`/` 以外が1文字以上」（URLの1区切り分）
  - `(?P<id>...)` … 当てはまった部分を **`id` という名前で取り出す**（名前付きキャプチャ）
- **このプロジェクト**：`/thread/{id}` → `#^/thread/(?P<id>[^/]+)$#` のように変換して使う。

### コンパイル（ここでの意味）
- **ざっくり**：`/thread/{id}` のような書き方を、**照合できる正規表現に変換**しておくこと。
- **詳しく**：ルート登録時に1回だけ変換し、結果を Route が保持。毎リクエストでは変換済みのものを使うので速い。
- **このプロジェクト**：[`Route::compile()`](../../src/Presentation/Routing/Route.php#L32)。

### 名前付きキャプチャ
- **ざっくり**：正規表現で当てはまった部分に**名前を付けて取り出す**書き方。
- **詳しく**：`(?P<id>[^/]+)` と書くと、一致した文字を `$m['id']` のように名前で受け取れる。番号で数えなくてよい。
- **このプロジェクト**：`{id}` がこれに変換され、`resolve()` が `params['id']` として取り出す。

### `array_reduce` / クロージャ（無名関数）
- **ざっくり**：**配列を畳み込んで1つにまとめる**関数。クロージャは「名前のないその場限りの関数」。
- **詳しく**：`array_reduce` でミドルウェア配列を順に包み込み、最終的に「1個の呼び出し可能な処理（パイプライン）」を作る。
- **このプロジェクト**：[`dispatch()`](../../src/Presentation/Routing/Router.php#L113) で玉ねぎ構造を組み立てるのに使用。

### O(1) / 計算量
- **ざっくり**：処理の速さの目安。**O(1)＝データ量が増えても一定速度**（速い）。
- **詳しく**：連想配列の一発引きは O(1)。一方、動的ルートを1本ずつ照合するのは本数に比例（遅め）。だから静的ルートを先に引く。
- **このプロジェクト**：[`resolve()` のコメント](../../src/Presentation/Routing/Router.php#L64)に「O(1) で先引き」とある。

## E. セキュリティ用語

### CSRF（クロスサイト・リクエスト・フォージェリ）
- **ざっくり**：利用者になりすまして**勝手にフォーム送信させる攻撃**。
- **詳しく**：対策として、フォームに秘密の合言葉（CSRFトークン）を埋め込み、送信時に一致するか確認する。一致しなければ拒否（403）。
- **このプロジェクト**：[`CsrfMiddleware`](../../src/Presentation/Routing/Middleware/CsrfMiddleware.php) が POST のとき `_csrf` を検証する。

### CSRFトークン
- **ざっくり**：本物の送信か確かめるための**使い捨ての合言葉**。
- **このプロジェクト**：`$_SESSION['_csrf']`（サーバー側）と送信値を突き合わせる。

### `hash_equals`
- **ざっくり**：文字列を**安全に比較する**関数。
- **詳しく**：普通の比較は「どこまで一致したか」が処理時間に出てしまい、それを手がかりに合言葉を当てられる恐れ（タイミング攻撃）がある。`hash_equals` は常に一定時間で比較してそれを防ぐ。
- **このプロジェクト**：CSRFトークンの照合に使用。

### XSS / エスケープ / `htmlspecialchars`
- **ざっくり**：**XSS** = 入力に紛れ込ませた悪意あるスクリプトを画面で実行させる攻撃。**エスケープ** = 危険な文字を無害な表記に置き換えること。
- **詳しく**：`<` を `&lt;` などに変換すれば、タグとして解釈されず文字として表示される。
- **このプロジェクト**：[`Response::error()`](../../src/Presentation/Http/Response.php#L38) がエラーメッセージを `htmlspecialchars` でエスケープしてから出力する。

### `APP_ENV`（development / production）
- **ざっくり**：いま「**開発中か本番か**」を示す設定。
- **詳しく**：開発中は詳しいエラーを画面に出して直しやすく、本番では中身を隠して情報漏れを防ぐ。
- **このプロジェクト**：500エラー時、[index.php](../../public/index.php#L30) が `APP_ENV` を見て表示内容を切り替える。

## F. インフラ用語

### nginx / Webサーバー
- **ざっくり**：外からのアクセスを最初に受け取る**玄関**ソフト。
- **このプロジェクト**：[docker/nginx](../../docker/nginx) の設定で、すべてのアクセスを `index.php` に集める。

### `try_files`
- **ざっくり**：nginx の指示で「**そのファイルが実在すれば返す、無ければ別のものへ回す**」。
- **このプロジェクト**：`try_files $uri $uri/ /index.php?$query_string;` で、実在しないURLをすべて `index.php` に回す（＝フロントコントローラーが成立する）。

### FastCGI / PHP-FPM
- **ざっくり**：nginx が受け取ったPHPの処理を、**PHP実行担当に橋渡しする**仕組み。
- **このプロジェクト**：設定の `fastcgi_pass php:9000;` がそれ。`php` という別コンテナでPHPを動かしている（[docker-compose.yml](../../docker-compose.yml) 参照）。
