## 主要 UseCase 詳細

### CreateThread

**入力**

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| title | string | Yes | スレッドタイトル |

**処理フロー**

```
1. タイトルのバリデーション（1〜100文字）
2. ThreadTitle VO 生成
3. Thread エンティティ生成（HP:100, 汚染:0, 変異:0）
4. ThreadRepository::save()
5. ThreadCreated ドメインイベント発火
6. 生成した Thread を返却
```

**出力**

Thread エンティティ

---

### CreatePost（最重要UseCase）

投稿作成。このUseCaseがゲームのメインループ。

**入力**

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| threadId | string | Yes | 投稿先スレッドID |
| content | string | Yes | 投稿内容 |
| ipAddress | string | Yes | 投稿者IP（ハッシュ生成用） |

**処理フロー**

```
1. スレッド取得（ThreadRepository::findById）
2. スレッド投稿可能チェック（canPost()）
   → 不可なら例外

3. 投稿内容バリデーション（1〜1000文字）
4. AuthorHash 生成（IP + Salt → SHA256）

5. お題ルール判定（ChallengeJudgeService）
   → 違反時：ペナルティ処理
     - hp_damage: スレHP減少
     - garble: 投稿内容を文字化けに変換
     - mute: 投稿を拒否（例外）
     - corruption_up: スレ汚染度上昇

6. Post エンティティ生成
7. PostRepository::save()

8. スレッド状態更新
   - HP を微減（-1〜-3、ランダム）
   - 汚染度を微増（+1〜+2、ランダム）

9. イベント判定（EventJudgeService）
   → イベント発生時：
     - イベント効果適用（スレへのダメージ、天候変化等）
     - GameEvent エンティティ生成・保存
     - 対応するドメインイベント発火

10. 世界状態更新
    - 危険度を微調整
    - フェーズ遷移チェック（WorldPhaseService）

11. PostCreated ドメインイベント発火
12. 結果を返却（Post + 発生イベント一覧）
```

**出力**

| フィールド | 型 | 説明 |
|---|---|---|
| post | Post | 作成された投稿 |
| events | GameEvent[] | 発生したイベント一覧 |
| thread | Thread | 更新後のスレッド状態 |
| worldState | WorldState | 更新後の世界状態 |

### CollapseThread

HP が 0 になったスレッドを崩壊させる。

**処理フロー**

```
1. スレッド取得
2. HP が 0 以下か確認
3. Thread::collapse() 実行
4. ThreadRepository::save()
5. ThreadCollapsed ドメインイベント発火
6. 世界の危険度を +5
```

---

### MutateThread

変異レベルが MAX になったスレッドを変異させる。

**処理フロー**

```
1. スレッド取得
2. 変異レベルが 5 か確認
3. Thread のステータスを mutated に変更
4. スレッド内の全投稿にランダムな変異を適用
   - 一部の投稿が文字化け
   - レスの順序がシャッフル
5. ThreadRepository::save()
6. MutationOccurred ドメインイベント発火
```

---

### TriggerWorldEvent

世界全体に影響するイベントを発動する。

**処理フロー**

```
1. 現在の WorldState 取得
2. イベント種別に応じた処理
   - 隕石: 全スレッドに HP -20
   - 赤い雨: 世界天候変更 + 全スレ汚染度 +10
   - 停電: 全スレ凍結（30秒間）
   - 汚染波: 全スレ汚染度 +20
3. WorldState 更新・保存
4. GameEvent 生成・保存
5. WorldChanged ドメインイベント発火
```

---

### ChangeWeather

世界の天候を変更する。

**処理フロー**

```
1. 現在の WorldState 取得
2. 新しい天候を設定
3. 天候に応じた追加効果
   - 赤い雨: 汚染系イベント発生率 UP
   - 嵐: ダメージ系イベント発生率 UP
   - 虚無: 全イベント発生率 DOWN（代わりに異常度 UP）
4. WorldStateRepository::save()
5. WeatherChanged ドメインイベント発火
```

---

### RotateChallenge

お題を新しいものに入れ替える。

**処理フロー**

```
1. 現在のお題を期限切れに
2. 世界フェーズに応じたお題プールから抽選
   - calm: 簡単なお題1つ
   - unstable: 普通のお題1つ
   - chaos: お題2つ同時
   - apocalypse: 難しいお題2つ同時
3. 新しい Challenge エンティティ生成（有効期間: 30〜60分）
4. ChallengeRepository::save()
5. ChallengeRotated ドメインイベント発火
```

---

### GetWorldState

現在の世界状態を取得する。ポーリング用。

**処理フロー**

```
1. WorldStateRepository::getCurrent()
2. WorldState を返却
```

---

### GetRecentEvents

直近のイベントを取得する。イベントログ表示用。

**入力**

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| limit | int | No | 取得件数（デフォルト20） |

**処理フロー**

```
1. EventRepository::findRecent($limit)
2. GameEvent[] を返却
```

## UseCase 依存関係図

```
CreatePost（メインループ）
  ├── ThreadRepository
  ├── PostRepository
  ├── WorldStateRepository
  ├── EventRepository
  ├── ChallengeRepository
  ├── EventJudgeService
  ├── ChallengeJudgeService
  ├── WorldPhaseService
  └── RandomGenerator（インターフェース）

CreateThread
  ├── ThreadRepository
  └── RandomGenerator

TriggerWorldEvent
  ├── WorldStateRepository
  ├── ThreadRepository（全スレ影響時）
  └── EventRepository

RotateChallenge
  ├── ChallengeRepository
  ├── WorldStateRepository（フェーズ参照）
  └── RandomGenerator
```
