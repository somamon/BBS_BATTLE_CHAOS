# アーキテクチャ設計

## レイヤードアーキテクチャ

```
  Presentation（HTTP、View）
       │
       ↓
    UseCase（アプリケーションロジック）
       │
       ↓
    Domain（ビジネスロジック）
       │
       ↓
  Infrastructure（DB、外部依存）
```

**依存の方向**: 外側 → 内側のみ。Domain は何にも依存しない。

## 依存関係ルール

| レイヤー | 依存可能な対象 | 依存してはいけない対象 |
|---|---|---|
| Domain | なし（自己完結） | UseCase, Infrastructure, Presentation |
| UseCase | Domain | Infrastructure, Presentation |
| Infrastructure | Domain（インターフェース実装） | UseCase, Presentation |
| Presentation | UseCase, Domain | Infrastructure（直接参照しない） |

---

## 依存注入（DI）

フレームワークなしのため、手動でDIを構成。

**エントリポイント（public/index.php）での組み立て**

```
1. Config 読み込み
2. DB接続生成（SQLiteConnection）
3. Repository 実装の生成
4. ドメインサービスの生成
5. イベントディスパッチャーの生成 + リスナー登録
6. UseCase の生成（依存を注入）
7. Controller の生成（UseCaseを注入）
8. Router にルート登録
9. リクエスト処理
```

将来的にはシンプルなDIコンテナを自作しても良い。

---

## リクエスト処理フロー

```
HTTP Request
  → public/index.php（フロントコントローラー）
    → Router::dispatch()
      → Middleware チェック（CSRF、RateLimit）
        → Controller::action()
          → UseCase::execute()
            → Domain ロジック実行
            → Repository 経由で永続化
            → ドメインイベント発火
          → View レンダリング（HTMLまたはHTML片）
        → HTTP Response
```

## Composer設定

PSR-4 オートロードのみ使用。

```json
{
    "autoload": {
        "psr-4": {
            "App\\": "src/"
        }
    },
    "require": {
        "php": ">=8.3"
    }
}
```

名前空間マッピング:

| ディレクトリ | 名前空間 |
|---|---|
| src/Domain/ | App\Domain\ |
| src/UseCase/ | App\UseCase\ |
| src/Infrastructure/ | App\Infrastructure\ |
| src/Presentation/ | App\Presentation\ |
| src/Shared/ | App\Shared\ |

## Viewレンダリング

フレームワークなしのため、素のPHPテンプレートを使用。

**レイアウト方式**

- `layout.php` に共通HTML（head, header, footer）
- 各ページは `$content` 変数でレイアウトに埋め込み
- HTMX部分更新用の `partial/` はレイアウトなしで返却

**出力エスケープ**

- 全出力は `htmlspecialchars()` でエスケープ
- ヘルパー関数 `e()` を定義して簡略化

## エラーハンドリング

| 種別 | 処理 |
|---|---|
| ドメイン例外 | UseCase内でcatch → ユーザー向けメッセージ返却 |
| バリデーション例外 | Controller でcatch → 422 エラーレスポンス |
| システム例外 | グローバルハンドラでcatch → 500 エラーページ → ログ記録 |
