# リデザイン2: 投稿に賭ける（目利きが儲かる掲示板）

> **本書が新しいコア設計の「正」。** [20_リデザイン_朽ちると投資.md](20_リデザイン_朽ちると投資.md) を次の一点で上書きする：
> **投資対象を「スレッド」から「投稿(post)」に移し、報酬を「配当」ではなく「株価上昇（ボンディングカーブ）」で与える。**
> 「朽ちる(HP減衰)」「世界フェーズ(相場天候)」は doc20 のまま**残す**（世界観の核）。
> doc20 §2.2 の配当モデル・§2.6 のスレ変異は v1 では**採用しない**（→ §9 ロードマップ）。

## 0. なぜ作り直すか（設計判断の記録）

旧コアは「面白い投稿が出る → スレに投資 → スレが延命する」で**終わって**いた。投資した後のドラマが無い。

新コアの一点突破ルール：

> **早く投資した人が得をする（目利きが主役）。**

いいね型は「みんなが良いと言った後」に押しても損しない（後追いOK）。
投資型に**早期報酬**を入れると、「みんなが気づく前に見抜く」固有の体験が生まれる。これは Reddit/5ch に無く、本サービスの差別化の核になる。

実現手段は **ボンディングカーブ（後から買うほど株価が高い）** 一本に絞る。これだけで
①投稿が育つ ②目利きが評価される ③資産ランキング が同時に成立する。
配当・生存予想・株主総会・ギルド等は**最初は入れない**（§9）。

## 1. コンセプト

```
┌─ 投資家層（登録ユーザーのみ）────────────────────┐
│  お金(初期500)で「投稿の株」を買う。後から買う人が増える     │
│  ほど株価が上がり、早く仕込んだ株が値上がりして資産が増える。   │
└──────────────────▲──────────────────────┘
                       │ 投資が「将来性シグナル」になり、投稿が育つ
┌─ 匿名掲示板層（誰でも）──┴──────────────────────┐
│  匿名でスレ立て・投稿（お金不要・報酬なし）。スレも投稿も       │
│  時間で朽ちて消える。投資の一部が投稿を延命させる。            │
└────────────────────────────────────────┘
```

**投資対象は「投稿(post)」**。スレッドは投稿を束ねる**板（コンテナ）**に格下げする。

中核ループ：

1. 匿名で投稿する（無料・無報酬）
2. 面白い投稿に**お金を賭ける＝株を買う**（要登録）
3. 後続の投資が増えるほど**株価が上がる**
4. 早く買った株が値上がりして**資産が増える**
5. 累計投資で投稿が**進化**（新規 → 注目 → 人気 → 殿堂入り）
6. 終局時の総資産（所持金 ＋ 保有株の評価額）でランキング

**ゴール（競争）**: 総資産ランキング上位 ＝ **良い投稿を早く見抜いた目利き**が勝つ。

## 2. 主要メカニクス

### 2.1 遅延減衰（Lazy Decay）— doc20 §2.1 を踏襲

cron を使わず、読むたびに経過時間から現在 HP を計算する。**投稿(post)にもスレ(thread)にも適用**。

```
現在HP = max(0, stored_hp − decay_per_min × phase_multiplier × 経過分数)
経過分数 = (now − last_decay_at) ÷ 60秒
phase_multiplier = 現在の世界フェーズの減衰倍率（§2.6。平常 1.0）
```

- 表示は計算のみ（書き込み不要）。投資など HP が動くイベント時に確定（stored_hp 書き戻し＋last_decay_at=now）。
- 現在HP ≤ 0 で `status='dead'`。dead は一覧から除外、**復活不可**、株は**紙くず（評価額0）**。
- スレが dead になると配下の投稿も dead（カスケード）。

### 2.2 投稿の株とボンディングカーブ — 経済の心臓

各**投稿**が「株価」を持つ。株価は**その投稿に集まった累計投資額が増えるほど上がる**。

```
株価(spot) = SHARE_PRICE_BASE + SHARE_PRICE_SLOPE × total_invested
```

- `total_invested` … その投稿に投じられた累計金額（円）。
- 後から投資するほど `total_invested` が大きく、株価が高い → **早期ほど安く多くの株を買える**。

### 2.3 投資の処理（1トランザクション）

登録ユーザーが投稿に `amount`（≥`MIN_INVEST`、所持金以下）を投資すると：

| 配分 | 割合 | 行き先 |
|---|---|---|
| 株取得 | **70%** | `0.7 × amount` で**現在株価から株を購入**。`shares = floor(0.7×amount ÷ 株価spot)` |
| HP回復(延命) | **30%** | 対象投稿の `hp += 0.3 × amount`（`max_hp` 上限、超過は捨てる＝ソフトsink） |

処理手順（すべて1トランザクション、失敗時ロールバック）：

1. 投稿が alive か・残高十分か確認（dead や残高不足は拒否）
2. 現在株価 `株価spot = BASE + SLOPE × total_invested` を算出
3. `shares = floor(0.7 × amount ÷ 株価spot)`（0株になる極小額は拒否）
4. 投稿を遅延減衰確定 → `hp += 0.3 × amount`（max_hp 上限）
5. `post.total_invested += amount`、`post.total_shares += shares`
6. `holdings(user, post).shares += shares`
7. `user.money −= amount`
8. レベル判定（§2.5）：`total_invested` で level 再計算、上がれば max_hp 引き上げ
9. `investments` に監査ログを追記

> **配当・sink・スレ変異は無し**（doc20 §2.2/§2.6 から削除）。早期報酬は**株価上昇のみ**で与える。
> HP に回した 30% は投資者の手元から消える（＝デフレ要因。§8 の有限ラウンド性を維持）。

### 2.4 株の評価額（マークトゥマーケット）

```
保有株の評価額 = my_shares × 株価spot × 鮮度
鮮度 = current_hp ÷ max_hp        （0〜1。朽ちるほど価値が目減り、dead で 0）
```

- **早期保有が値上がり**: 後続投資で `総invested` が増え `株価spot` が上がる → 早く買った株の評価額が伸びる。
- **朽ちると価値が溶ける**: 放置で HP が減ると `鮮度` が下がり評価額が落ちる。dead で 0。
- 購入は `株価spot`（鮮度を掛けない素の株価）で行い、評価のみ鮮度を掛ける。
  → **朽ちかけの投稿を買うと即含み損**になり、安易な後追いを抑制。延命投資で蘇生を狙う逆張りは可能。

**目利きが儲かる例**（BASE=10, SLOPE=0.01, 株取得70%）：

```
投稿「宇宙人を見た」 新規（total_invested=0, 株価¥10）
  Alice が ¥100 投資 → 0.7×100=¥70 で 7株取得 / ¥30 がHP回復
  → total_invested=100, 株価=10+100×0.01=¥11, レベル→「注目」
  その後、他人の投資が合計 ¥9,900 集まる → total_invested=10,000
  → 株価=10+10000×0.01=¥110, レベル→「殿堂入り」
  → Alice の 7株 = 7 × ¥110 × 鮮度1.0 = ¥770（投資¥100の約7.7倍）
```

後から買う人は株価¥110で買うので、同じ¥100でも 0.6株しか取れず旨味が薄い
→ **「まだ誰も気づいていない投稿」を探す動機**（埋もれた名作の発掘）が自然に生まれる。

### 2.5 投稿レベル（成長の可視化）

`total_invested` の閾値で投稿が進化する。レベルは**ステータス表示**＋**max_hp（耐久）**を強化する。

| レベル | 必要 total_invested | max_hp | 演出 |
|---|---|---|---|
| 新規 | 0 | 100 | 通常 |
| 注目 | 100 | 300 | 「注目」バッジ |
| 人気 | 1,000 | 800 | タイトル強調 |
| 殿堂入り | 10,000 | 2,000 | 「殿堂入り」化・長寿命のランドマーク |

- レベルが上がると max_hp が増え、**人気投稿ほど朽ちにくくなる**（名作が残る）。
- レベル判定は投資トランザクション内（イベント駆動）。閾値到達で level 更新 ＆ max_hp 引き上げ。

### 2.6 世界フェーズ（相場天候）— doc20 §2.5 を踏襲（変更なし）

世界に1行の `world_state` を持ち、全 HP 減衰に倍率を掛ける。遅延遷移・cron 不要。

| フェーズ | 倍率 | 体感 |
|---|---|---|
| `boom` | ×0.7 | 朽ちにくい・強気相場 |
| `calm` | ×1.0 | 通常 |
| `storm` | ×1.8 | 急速に朽ちる・延命コスト増 |
| `crash` | ×2.5 | 投げ売り・投資妙味は最大 |

## 4. DB 設計（MySQL 8.0）— doc20 §4 からの差分

要点：**株情報・レベルを threads から posts へ移す**。holdings / investments の参照先を thread → post に変更。

### threads（板＝コンテナに格下げ）
- **残す**: id, creator_id, title, hp, max_hp, decay_per_min, last_decay_at, status, post_count, created_at, updated_at
- **削除**: `mutation_level`, `total_shares`（株はスレが持たない）
- 索引: `idx_threads_status_created (status, created_at)` 据え置き

### posts（投資対象に昇格）
- **残す**: id, thread_id, author_hash, author_id, content, hp, decay_per_min, last_decay_at, status, created_at
- **追加**:

| カラム | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| max_hp | INT | NO | 100 | HP上限（レベルで上昇） |
| total_invested | BIGINT | NO | 0 | 累計投資額（株価・レベルの基礎） |
| total_shares | INT | NO | 0 | 発行済み総株数 |
| level | TINYINT | NO | 0 | 0=新規 / 1=注目 / 2=人気 / 3=殿堂入り |
| updated_at | DATETIME | NO | — | 更新時刻（株/HP確定で更新） |

- 索引追加: `idx_posts_status_invested (status, total_invested)`（人気順・ランキング集計用）

### holdings（持ち分）— **thread_id → post_id に変更**

| カラム | 型 | NULL | 説明 |

| user_id | VARCHAR(26) | NO | 株主 (FK users) |

**PK**: (user_id, post_id) / **索引**: `idx_holdings_post (post_id)`

### investments（監査ログ）— **thread_id → post_id、配分内訳を更新**

| カラム | 型 | NULL | 説明 |

| id | VARCHAR(26) | NO | ULID (PK) |

**索引**: `idx_investments_investor (investor_id)`, `idx_investments_post (post_id)`

### world_state — doc20 §4 のまま（変更なし）

> **マイグレーション戦略**: 旧 004/005（holdings/investments の thread_id 版）は作り直す。既存データは無いので破壊的変更で可（新規マイグレーション 007 以降で posts へカラム追加・holdings/investments を post 参照に張り替え）。実装フェーズで採番。

## 7. アーキテクチャ方針（クリーンアーキテクチャ維持）

現行の層構成（Domain / Application / Infrastructure / Presentation）をそのまま使う。差分は局所的：

- **Domain**
  - `Post` エンティティに株・株価・レベル・HPロジックを集約（`currentHp()`, `settleDecay()`, `buyShares()`, `valuation()`, `levelFor()`）。
  - `Thread` から株・変異ロジックを除去（板の HP/decay のみ）。
  - `HoldingRepository` / `InvestmentRepository` の対象を post に。
- **Application**
  - `InvestInThread` → `InvestInPost` に作り替え（配当配分を削除、株価購入＋HP回復＋レベル判定）。
  - `RankingQuery` / `MyPageQuery` の評価額計算を §2.4 の式に差し替え。
  - 株価・鮮度の計算は Domain（`Post`）に置き、UseCase は呼ぶだけ（二重実装防止）。
- **Infrastructure**: `PdoPostRepository` に株/レベル列の read/write を追加。holdings/investments の SQL を post 参照に。
- **Presentation**: 投資フォームを投稿カード単位に。ルート `/post/{id}/invest` 追加。
- ゲーム定数は `App\Config\Game` に一元化（§3）。

## 8. 経済特性

- レス報酬 0・配当なし。投資のたびに HP に回した 30%（max_hp 超過分含む）が循環マネーから退場 ＝ **デフレ**。
- お金の天井は「初期500 × 登録人数」。投資が進むほど現金が株（紙の評価額）に変わり、現金は減る。
- **有限ラウンド**（doc20 §8 を踏襲）：現金が枯れ、投稿が朽ちて評価額が溶け、終局へ向かう。

**終局条件**（どちらか・遅延チェックで判定）：
- 全スレッドが dead、または
- 全登録ユーザーの所持金合計 < `MIN_INVEST`

**終局処理**: ランキング確定 → リザルト画面で勝者（総資産トップ＝最高の目利き）を表示。
