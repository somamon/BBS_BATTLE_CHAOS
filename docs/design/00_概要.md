# 異常掲示板 — プロジェクト概要

## コンセプト

「掲示板を作る」のではなく「異常な世界を作る」。
掲示板UIは、その世界へのインターフェースに過ぎない。

## 仮タイトル

**異常掲示板**

## コア体験

ユーザーが感じるべきもの：

- 「この掲示板、生きてる…」
- 「何が起こるかわからない」
- 「自分の投稿で世界が変わった」
- 「ルールを完全に把握できない」
- 「深夜に変な現象が起きる」

## 掲示板の特異性

| 普通の掲示板 | 異常掲示板 |
|---|---|
| スレッドは静的データ | スレッドは生物（HPあり、死ぬ） |
| 投稿は記録 | 投稿はトリガー（世界が変わる） |
| UIは固定 | UIが侵食される |
| 時間は無関係 | 深夜に異変が起きる |
| ルールは明確 | ルールが変異する |

## 体験フロー

```
ユーザーがスレを開く
  → スレにHPバーと汚染ゲージがある
  → 投稿する
  → 乱数判定が走る
  → 「隕石が落ちました。スレHPが30減少」
  → 世界の天候が「赤い雨」に変化
  → 画面が赤くなる
  → 他のスレにも影響が波及
  → 別のユーザーが「何が起きた？」と投稿
  → その投稿でさらにイベント発生
  → 連鎖
```

## 技術スタック

| レイヤー | 技術 |
|---|---|
| バックエンド | PHP 8.3（バニラ、フレームワークなし） |
| フロント | SSR + HTMX + Alpine.js + CSS Animation |
| DB（初期） | SQLite |
| DB（将来） | MySQL |
| パッケージ管理 | Composer（オートロード用） |

## MVP範囲

最小限で動くものを先に作る。

### 必須（Phase 1）

- スレッド作成・表示
- 投稿
- 世界状態管理

### 次点（Phase 2）

- ランダムイベント
- スレッド状態変化（HP、汚染）
- お題システム

### 演出（Phase 3）

- UI侵食
- 特殊演出
- 深夜イベント
- リアルタイム更新強化
