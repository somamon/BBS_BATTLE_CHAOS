# 要件定義

> 本書のゲーム内容は [20_リデザイン_朽ちると投資.md](20_リデザイン_朽ちると投資.md) を正典とする。
> メカニクス・スキーマ・定数・名称はすべて正典に準拠し、超える仕様は将来拡張（§スコープ外）に置く。

## 0. 概要

**2層構造の掲示板ゲーム。**

- **匿名掲示板層（誰でも）**: 匿名でスレ立て・レス投稿（お金不要・報酬なし）。スレもレスも個別 HP を持ち、時間で朽ちて 0 で消滅する。
- **投資家層（登録ユーザーのみ）**: email + password で登録すると初期所持金 500 を持ち、スレッドの「株」を買える。配当と株評価益で総資産を競う。

技術基盤: 自作ルーティング / Plates + PHP-DI / 簡易MVC（`App\Model` + `Presentation/Controller|View`）/ ゲーム定数は `App\Config\Game` に集約 / MySQL 8.0 / Docker。POST 系は CSRF 必須、サーバサイドで HTML を組み立てて返す。

## 1. 機能要件（FR）

各 FR に「概要 / 主な振る舞い / 受入条件」を記す。認証区分は **匿名可** / **要登録** を明記する。

### FR-01: 匿名スレ作成（匿名可）

- **概要**: 誰でも匿名でスレッドを作成できる。お金・報酬は発生しない。
- **主な振る舞い**: タイトルを受け取り thread を作成する。HP は満タン（`THREAD_INIT_HP` = 300、`max_hp` = `THREAD_MAX_HP` = 1000）、`total_shares=0`、`mutation_level=0`、`status='alive'`、`last_decay_at=now`。登録ユーザーが作成した場合は `creator_id` に user_id を記録（表示用、匿名は NULL）。
- **受入条件**:
  - 未登録でもスレを作成できる。
  - 作成直後のスレは HP 満タンで一覧に表示される。

### FR-02: 匿名レス投稿（匿名可）

- **概要**: 誰でも匿名でスレッドにレスを投稿できる。お金・報酬は発生しない（無料・無報酬）。
- **主な振る舞い**: 対象スレが `alive` であることを確認 → post を作成（`POST_INIT_HP` = 100、`status='alive'`、`last_decay_at=now`、`author_hash` に IP 由来ハッシュ）→ thread の `post_count++`。登録ユーザーの投稿は `author_id` を記録（表示用）。
- **受入条件**:
  - 未登録でもレスを投稿できる。
  - dead のスレにはレスを投稿できない。
  - 投稿後、対象スレの `post_count` が増える。

### FR-03: ユーザー登録（不要 → 登録）

- **概要**: email + name + password で投資家として登録する。
- **主な振る舞い**: email / name / password を受け取り、`password_hash()` で password_hash を生成 → users を作成（`money=INITIAL_MONEY`=500）→ セッションに user_id を紐付ける。
- **受入条件**:
  - email は一意。重複時はエラー。
  - 形式不正（email 形式・必須欠落）はエラー。
  - 登録成功で所持金 500 が付与される。
  - パスワードは平文保存せず `password_hash()` の出力のみを保存する。

### FR-04: ログイン（匿名可＝認証不要で実行）

- **概要**: email + password でログインする。
- **主な振る舞い**: email で users を引き → `password_verify()` で照合 → 一致でセッションに user_id を保存。
- **受入条件**:
  - メール不在 / パスワード不一致はエラー（どちらかを過度に特定しない）。
  - ログイン後、要登録機能（投資・マイページ）が利用できる。

### FR-05: 遅延減衰によるHP表示と消滅（匿名可）

- **概要**: cron を使わず、**読むたびに**経過時間から現在 HP を計算して表示する。0 で消滅（dead）。
- **主な振る舞い**:
  - `現在HP = max(0, stored_hp − decay_per_min × phase_multiplier × 経過分数)`、`経過分数 = (now − last_decay_at) ÷ 60秒`。
  - `phase_multiplier` は現在の世界フェーズ倍率（FR-09、平常は 1.0）。
  - 表示は計算のみ（書き込み不要）。HP が変わるイベント（投資等）時に「現在HP算出 → stored_hp に書き戻し → `last_decay_at=now`」で確定。
  - 現在HP ≤ 0 で `status='dead'`。dead は一覧から除外し、復活不可。
- **受入条件**:
  - 一覧・詳細で各スレ／レスの現在 HP が経過時間に応じて減って見える。
  - HP が 0 以下のスレ／レスは一覧に出ず、再表示されない。
  - 減衰は世界フェーズ倍率を反映する。

### FR-06: スレへの投資（要登録）

- **概要**: 登録ユーザーがスレッドに `amount`（≥ `MIN_INVEST`=1、所持金以下）を投資する。投資額は HP回復50% / 配当30% / 消滅(sink)20% に分かれ、投資者は `amount` 株を取得する。
- **主な振る舞い**（1トランザクションで実行、失敗時ロールバック）:
  1. 残高・スレ `alive` を確認。
  2. 投資者の所持金 `−amount`。
  3. **HP回復 50%**: 対象スレの `hp += SPLIT_HP × amount`（= 0.5×amount）。`max_hp` 超過分は sink へ。
  4. **配当 30%**: 新規株を発行する **前** の既存株主へ持ち分比率で分配（投資者本人は対象外）。既存株主が居なければ（＝最初の投資者）その回の配当は sink に合流。変異スレは配当に `配当ボーナス`（FR-08）を掛け、その分 sink を減らす。
  5. **sink 20%**: 経済から消す。
  6. 株発行: `holdings` に `amount` 加算、`thread.total_shares += amount`。
  7. 変異判定（FR-08）。
  8. investments に監査ログ（amount / to_hp / to_dividend / to_sink）を記録。
- **受入条件**:
  - 未登録 / 残高不足 / dead / `amount < MIN_INVEST` は失敗。
  - 投資成功で投資者の保有株（holdings.shares）と `thread.total_shares` が `amount` 増える。
  - 最初の投資では配当が sink に合流し、既存株主への配当は発生しない。
  - 失敗時は所持金・HP・株・ログのいずれも変化しない（原子性）。

### FR-07: 株保有（holdings）と評価額（マイページは要登録 / 表示は匿名可）

- **概要**: 保有株とその評価額を扱う。
- **主な振る舞い**: `保有株の評価額 = (my_shares ÷ thread.total_shares) × スレの現在HP`。現在HP は遅延減衰で算出する。
- **受入条件**:
  - 保有株ごとに現在 HP に基づく評価額が算出される。
  - スレが dead（現在HP=0）になると評価額は 0（株は紙くず）。

### FR-08: 変異（化け株）（投資の一部として駆動）

- **概要**: スレの累計投資 `total_shares` が閾値（`MUTATION_TIERS` = 500 / 2000 / 8000）を超えると変異し、`mutation_level`（0〜3）が上がって `max_hp` 上昇と配当ボーナス強化を得る。
- **主な振る舞い**: 投資トランザクション内で判定。閾値到達で `mutation_level++`、`max_hp` を `MUTATION_MAX_HP`（2000 / 4000 / 8000）へ引き上げ。配当ボーナス `MUTATION_DIV_BONUS`（1.1 / 1.2 / 1.3）を以降の新規投資の配当配分に適用。
- **受入条件**:
  - `total_shares` が閾値を超えた投資で `mutation_level` が 1 段階上がり、`max_hp` が更新される。
  - 変異スレへの新規投資で配当が配当ボーナス分だけ厚く分配され、その分 sink が減る。
  - レスは投資対象外なので変異・株を持たない（HP が減るのみ）。

### FR-09: 世界フェーズ表示と減衰倍率（表示は匿名可 / 遷移は遅延）

- **概要**: 世界に1行の `world_state` を持ち、全スレの減衰に倍率を掛ける「相場環境」を表す。
- **主な振る舞い**:
  - フェーズと倍率: `boom`×0.7 / `calm`×1.0 / `storm`×1.8 / `crash`×2.5。
  - **遅延遷移**: 任意アクセス時に `now ≥ next_shift_at` ならフェーズを抽選し直し、`phase` / `phase_multiplier` / `next_shift_at` を更新（1フェーズの継続秒は `PHASE_MIN_SEC`=180 〜 `PHASE_MAX_SEC`=900 の抽選）。cron 不要。
  - 遅延減衰は「現在フェーズ倍率 × 経過分」で近似（フェーズ切替時に全スレ書き戻しはしない）。
- **受入条件**:
  - トップ／一覧ヘッダーに現在フェーズが常時表示される。
  - `now ≥ next_shift_at` のアクセスでフェーズが再抽選され、`next_shift_at` が更新される。
  - 現在フェーズ倍率が FR-05 の減衰計算に反映される。

### FR-10: 資産ランキング（匿名可）

- **概要**: 全登録ユーザーの総資産で降順ランキングを表示する。
- **主な振る舞い**: 各ユーザーの `総資産 = money + Σ(保有株の評価額)` を集計し降順表示。評価額は遅延減衰で算出した現在 HP に基づく。
- **受入条件**:
  - 全ユーザーが総資産降順で並ぶ。
  - 表示名（name）でユーザーが識別できる。

### FR-11: マイページ（要登録）

- **概要**: ログインユーザーの所持金・保有株・各評価額・含み損益を表示する。
- **主な振る舞い**: 所持金、保有スレごとの保有株数・現在評価額・含み損益を一覧表示する。
- **受入条件**:
  - 未登録ではアクセスできない。
  - 保有株ごとに現在評価額と含み損益が表示される。

### FR-12: 有限ラウンド終局判定とリザルト表示（匿名可）

- **概要**: レス無報酬＋配当再分配（新規マネー無し）で経済はデフレ。投資のたびに 70%（HP化50%＋sink20%）が循環マネーから退場するため、本ゲームは構造上「有限ラウンド」。終局で勝者を表示する。
- **主な振る舞い**:
  - **終局条件（どちらか）**: 全スレッドが `dead`、または 全登録ユーザーの所持金合計 < `MIN_INVEST`（誰も投資できない）。
  - 終局判定は投資／閲覧のたびに軽くチェックする遅延方式（cron 不要）。
  - **終局処理**: ランキングを確定し、リザルト画面で勝者（総資産トップ）を表示。以降は新ラウンド開始（全データをアーカイブ → 初期化）まで閲覧のみ。
- **受入条件**:
  - 終局条件成立後、投資が受け付けられず閲覧のみになる。
  - リザルト画面に総資産トップが勝者として表示される。

## 2. 非機能要件（NFR）

### NFR-01: パフォーマンス・低負荷（遅延減衰）

- HP 減衰・世界フェーズ遷移・終局判定はすべて **遅延方式**（アクセス時計算）で行い、cron や全行更新を使わない。
- 表示時の減衰は計算のみで書き込みを伴わない。HP 変動イベント時のみ stored_hp を確定書き込みする。

### NFR-02: セキュリティ

- **CSRF**: すべての POST（スレ作成・レス投稿・投資・登録・ログイン）でトークン検証を必須とする。
- **パスワード**: `password_hash()` で保存、`password_verify()` で照合。平文を保存しない。
- **XSS**: ユーザー入力（タイトル・本文・表示名等）は出力時に HTML エスケープする。
- **SQLインジェクション**: プリペアドステートメントを使用する。

### NFR-03: データストア

- MySQL 8.0 を使用する。スキーマは正典 §4（users / threads / posts / holdings / investments / world_state）に準拠。
- `world_state` は常に1行（`id=1, phase='calm'`）を初期 INSERT する。
- ID は ULID（VARCHAR(26)）。

### NFR-04: 実行基盤・整合性

- Docker 上で動作する。単一サーバ運用可能。
- 投資は1トランザクション（出金・配当入金・HP更新・株発行・ログ）で実行し、失敗時はロールバックして原子性を保つ。
- 減衰計算は Model に集約（`Thread::currentHp()` / `decayAndPersist()` 等）して二重実装を防ぐ。投資ロジックは単一ユースケース（`InvestService::invest()`）にまとめ、トランザクション境界をそこに置く。

### NFR-05: レスポンス形式・保守性

- レスポンスはサーバサイドで組み立てた HTML（簡易MVC + Plates ビュー）。
- ゲーム定数は `App\Config\Game` に一元化し、バランス調整を集中管理する（正典 §3）。

## 3. スコープ外（将来拡張）

正典 §10 に従い、以下はコアに含めず将来拡張として扱う。コアを汚さず「相場イベント層」として段階的に追加する。

| 拡張 | 旧要素 | 投資ゲームでの意味 |
|---|---|---|
| 汚染（ジャンク化） | corruption | dead 接近や特定条件で減衰加速＆配当率低下。優良株／不良株の選別軸 |
| お題（テーマ株） | challenges | 定期お題で該当スレの減衰緩和・配当ボーナス。物色セクター |
| 連鎖イベント（市場サプライズ） | events | 投稿／投資起点で確率的に暴落・高騰・災害。イベントログ＝市場ニュース |
| 株の売却 / 二次流通 | — | 保有株の売却・株主間取引（本ラウンドでは買い増しと配当回収のみ） |

> 旧テーマ語（汚染 / お題 / 天候単体 / danger_level / anomaly_level 等）はコア要件には含めない。採用済みのコア相乗要素は **世界フェーズ（FR-09）** と **変異（FR-08）** の2つ。

### 実装時チューニング項目（残課題）

正典 §9 より、コア設計は確定。残るは実装時の調整項目のみ。

- 数値バランス（減衰・初期HP・配分%）は実プレイ後に `App\Config\Game` で調整。
- スレ作成のスパム対策（クールダウン等）は実装後に判断、MVP は無対策。
- レス延命の UI（勢いの見せ方：レス数・残HPバー等）は実装時に検討。
