# 要件定義

## 機能要件

### FR-01: スレッド機能

ユーザーがスレッドを作成・閲覧できる。

**スレッド属性**

| 属性 | 型 | 説明 |
|---|---|---|
| id | string (ULID) | 一意識別子 |
| title | string | スレタイトル（最大100文字） |
| hp | int | 体力（初期値100、0で崩壊） |
| corruption | int | 汚染度（0〜100、高いほど異常） |
| mutation_level | int | 変異レベル（0〜5） |
| weather | string | スレ固有天候 |
| status | string | 状態（active / collapsed / mutated / frozen） |
| created_at | datetime | 作成日時 |

**特殊仕様**

- スレッドは投稿により状態が変化する
- HP が 0 になるとスレッドは「崩壊」状態になる
- 汚染度が高いとイベント発生率が上がる
- 変異レベルが上がるとスレの挙動が予測不能になる

**スレッド状態遷移**

```
active（通常）
  ├─→ collapsed（HP 0、崩壊）
  ├─→ mutated（変異レベル MAX）
  ├─→ frozen（世界イベントにより凍結）
  └─→ active（回復イベント）
```

### FR-02: 投稿機能

ユーザーがスレッドにレス投稿できる。

**投稿属性**

| 属性 | 型 | 説明 |
|---|---|---|
| id | string (ULID) | 一意識別子 |
| thread_id | string | 所属スレッドID |
| content | string | 本文（最大1000文字） |
| author_hash | string | 投稿者識別ハッシュ（IP等から生成） |
| is_corrupted | bool | 汚染済みフラグ |
| created_at | datetime | 投稿日時 |

**投稿時処理フロー**

```
投稿受付
  → バリデーション
  → お題ルール判定
    → 違反時：ペナルティ処理
  → 投稿保存
  → 乱数抽選（イベント判定）
  → スレッド状態更新（HP、汚染度等）
  → 世界イベント判定
  → PostCreated イベント発火
  → レスポンス返却
```

### FR-03: イベント機能

投稿・時間経過によりイベントが発生する。

**イベントカテゴリ**

| カテゴリ | イベント例 | 影響範囲 |
|---|---|---|
| 天災 | 隕石、赤い雨、地震 | スレ単体 or 世界全体 |
| 異常 | 文字化け、時間逆行、スレ分裂 | スレ単体 |
| 復活 | 古いレス復活、崩壊スレ蘇生 | スレ単体 |
| システム | 停電、通信障害 | 世界全体 |
| 呪い | 投稿者呪い、連鎖汚染 | 投稿者 or スレ |

**発生条件**

| 条件 | 説明 |
|---|---|
| ランダム | 投稿時に確率で発生 |
| 特定単語 | 投稿内容に特定キーワードが含まれる |
| 深夜帯 | 0:00〜5:00 に発生率上昇 |
| 汚染度 | 汚染度が閾値を超えると発生 |
| 投稿数 | 累計投稿数が閾値を超えると発生 |
| 世界状態 | 世界の危険度・フェーズに応じて変化 |

### FR-04: 世界状態システム

掲示板全体が1つの「世界」として状態を持つ。

**世界状態属性**

| 属性 | 型 | 説明 |
|---|---|---|
| weather | string | 世界天候（晴れ / 赤い雨 / 霧 / 嵐 / 虚無） |
| danger_level | int | 危険度（0〜100） |
| anomaly_level | int | 異常度（0〜100） |
| phase | string | フェーズ（calm / unstable / chaos / apocalypse） |
| updated_at | datetime | 最終更新日時 |

**世界状態の影響**

| 状態 | 影響 |
|---|---|
| 天候変化 | UIの背景・色調が変化 |
| 危険度上昇 | イベント発生率が上昇 |
| 異常度上昇 | 投稿内容が自動変異する確率上昇 |
| フェーズ進行 | 新しいイベントが解禁される |

**フェーズ遷移**

```
calm（平穏）
  → unstable（不安定）  ※危険度 30 以上
    → chaos（混沌）     ※危険度 60 以上
      → apocalypse（終末）※危険度 90 以上
        → calm（リセット）  ※終末イベント完了後
```

### FR-05: お題システム

一定時間ごとに掲示板全体のルールが変更される。

**お題例**

| お題 | 制約 | 違反ペナルティ |
|---|---|---|
| 漢字禁止 | 漢字を含む投稿不可 | スレHP -10 |
| 5文字制限 | 5文字以内の投稿のみ | 投稿が文字化け |
| 語尾固定 | 指定語尾をつける | 投稿者ミュート（1分） |
| 嘘しか言えない | （判定不能なのでフレーバー） | なし |
| 敬語強制 | 「です」「ます」必須 | 汚染度 +5 |

**お題ローテーション**

- 30分〜1時間ごとに自動変更
- 世界フェーズによってお題の難易度が変化
- 複数お題が同時発動する場合あり（chaos以上）

### FR-06: リアルタイム更新

画面が自動的に最新状態を反映する。

**更新方式（初期）**

HTMX ポーリング（3秒間隔）

**更新対象**

| 対象 | エンドポイント | 間隔 |
|---|---|---|
| スレッド状態（HP等） | GET /api/thread/{id}/state | 3秒 |
| 世界状態 | GET /api/world/state | 3秒 |
| イベントログ | GET /api/events/recent | 3秒 |
| お題 | GET /api/challenge/current | 10秒 |

**将来拡張**

SSE → WebSocket へ段階的に移行可能な設計にする。

## 非機能要件

### NFR-01: パフォーマンス

| 項目 | 目標値 |
|---|---|
| 同時接続数 | 100接続 |
| スレッド更新 | 3秒以内 |
| 投稿応答 | 1秒以内 |
| ページ表示 | 2秒以内（初回） |

### NFR-02: 可用性

| 項目 | 内容 |
|---|---|
| サーバ構成 | 単一サーバ運用可能 |
| DB | SQLite で運用可能（将来 MySQL 移行可能） |
| バックアップ | SQLite ファイルの定期コピーで対応 |

### NFR-03: 保守性

| 項目 | 内容 |
|---|---|
| アーキテクチャ | クリーンアーキテクチャ採用 |
| ドメイン分離 | ビジネスロジックをドメイン層に集約 |
| イベント駆動 | イベント発火→リスナー実行の疎結合設計 |
| テスタビリティ | リポジトリインターフェースによるモック差替え可能 |

### NFR-04: セキュリティ

| 項目 | 対策 |
|---|---|
| XSS | 出力時HTMLエスケープ |
| SQLインジェクション | プリペアドステートメント使用 |
| CSRF | トークン検証（POST時） |
| レートリミット | 同一IPからの連続投稿制限 |
