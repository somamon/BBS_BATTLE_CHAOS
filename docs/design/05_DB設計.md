# DB設計

## 概要

- 初期: SQLite
- 将来: MySQL 移行可能
- SQLite/MySQL 両対応のため、型は共通的なものを使用
- リポジトリインターフェースでDB実装を隠蔽

---

## テーブル定義

### threads

スレッド情報。

| カラム | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | TEXT | NO | — | ULID (PK) |
| title | TEXT | NO | — | スレッドタイトル |
| hp | INTEGER | NO | 100 | 体力（0〜100） |
| corruption | INTEGER | NO | 0 | 汚染度（0〜100） |
| mutation_level | INTEGER | NO | 0 | 変異レベル（0〜5） |
| weather | TEXT | NO | 'sunny' | スレ固有天候 |
| status | TEXT | NO | 'active' | 状態（active/collapsed/mutated/frozen） |
| post_count | INTEGER | NO | 0 | 投稿数（キャッシュ用） |
| created_at | TEXT | NO | — | ISO 8601 形式 |
| updated_at | TEXT | NO | — | ISO 8601 形式 |

**インデックス**

| インデックス名 | カラム | 用途 |
|---|---|---|
| idx_threads_status | status | 状態別一覧取得 |
| idx_threads_created_at | created_at | 新着順一覧 |

## マイグレーション方針

- SQLite: DDL を PHP スクリプトで実行
- マイグレーションファイルは `database/migrations/` に連番管理
- 実行済みマイグレーションは `migrations` テーブルで記録

### migrations（管理用）

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER | 自動採番 (PK) |
| name | TEXT | マイグレーションファイル名 |
| executed_at | TEXT | 実行日時 |

## ER図（テキスト）

```
threads ──< posts
  │
  │ (target_id)
  ↓
events

world_states (単独・1レコード)

challenges (単独)
```

**リレーション**

| 関係 | 説明 |
|---|---|
| threads → posts | 1:N（スレッドは複数投稿を持つ） |
| threads → events | 1:N（target_id で紐付く、NULLの場合は世界イベント） |

## SQLite 固有の注意点

| 項目 | 対策 |
|---|---|
| 同時書き込み | WALモード有効化（`PRAGMA journal_mode=WAL`） |
| 外部キー制約 | `PRAGMA foreign_keys=ON` で有効化 |
| 日時型 | TEXT で ISO 8601 文字列として保存 |
| Bool型 | INTEGER（0/1）で代替 |
| JSON型 | TEXT として保存、PHP側でエンコード/デコード |

---

## MySQL 移行時の変更点

| 項目 | SQLite | MySQL |
|---|---|---|
| ID型 | TEXT | VARCHAR(26) |
| 日時型 | TEXT (ISO 8601) | DATETIME |
| Bool型 | INTEGER (0/1) | TINYINT(1) |
| JSON型 | TEXT | JSON |
| 自動採番 | AUTOINCREMENT | AUTO_INCREMENT |

リポジトリインターフェースにより、移行時はInfrastructure層の実装差替えのみで対応。
