# DB設計

> コア設計の正典は [20_リデザイン_朽ちると投資.md](./20_リデザイン_朽ちると投資.md) §4。
> 本書はその §4 スキーマを DB 設計書として転記・整形したもの。型・NULL・デフォルト・索引・PK は doc20 §4 と完全一致させる。

---

## 概要

- 採用DB: **MySQL 8.0**（docker-compose の `db` サービス。確定）
- エンジン: **InnoDB** / 文字コード: **utf8mb4**（`utf8mb4_0900_ai_ci`）
- 接続: PDO + `pdo_mysql`
- 型方針:
  - ID = **ULID** を `VARCHAR(26)`（PK）
  - 日時 = `DATETIME`
  - Bool = `TINYINT(1)`
- **遅延減衰（Lazy Decay）前提のためバッチ／cron は不要**。HP やフェーズは「読むたびに経過時間から算出 → 確定時に書き戻し」で確定する（doc20 §2.1 / §2.5）。`stored_hp`／`last_decay_at` がスナップショット列。
- DDL は `database/migrations/` に連番SQLで管理。初回起動時は db コンテナの `/docker-entrypoint-initdb.d` 経由で自動適用される。

> 本設計の初期案は SQLite だったが、実インフラ（MariaDB → MySQL 8.0）に合わせて MySQL を正式ターゲットに確定済み。リデザイン（doc20）に伴い旧 `threads`／`posts` は耐久スナップショット・株情報・変異を持つ新スキーマに**作り直す**。旧テーマ列（corruption / weather / danger_level / anomaly_level など）および旧 `events`／`challenges`／`world_states`（複数形）テーブルはコアから除外（将来拡張は doc20 §10）。

### world_state（世界フェーズ・常に1行）

世界に1行だけ存在し、全スレの減衰倍率（相場天候）を表す（doc20 §2.5）。複数形ではなく**単数 `world_state`**。

| カラム | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | TINYINT | NO | 1 | 固定値1 (PK) |
| phase | VARCHAR(10) | NO | 'calm' | boom / calm / storm / crash |
| phase_multiplier | DECIMAL(3,1) | NO | 1.0 | 現フェーズの減衰倍率 |
| next_shift_at | DATETIME | NO | — | 次にフェーズ抽選する時刻（遅延遷移） |
| updated_at | DATETIME | NO | — | 更新日時 |

## 初期データ

`world_state` に固定1行を投入する（`id=1, phase='calm'`、倍率1.0）。

```sql
INSERT INTO world_state (id, phase, phase_multiplier, next_shift_at, updated_at)
VALUES (1, 'calm', 1.0, NOW(), NOW());
```

> users / threads / posts / holdings / investments の初期データは無し（実行時に生成）。

## マイグレーション方針

- DDL は `database/migrations/` に**連番SQL**（例: `001_create_users.sql`, `002_create_threads.sql` …）で管理する。
- 初回起動時は db コンテナの **`/docker-entrypoint-initdb.d`** に連番SQLをマウントし、自動適用する（手動実行不要）。
- リデザイン（doc20）に伴い**スキーマを刷新**する。既存 `database/migrations/` には旧 `001_create_threads.sql` / `002_create_posts.sql`（旧2テーブル）のみが存在するが、これらは新スキーマ（耐久スナップショット・株情報・変異・`world_state`／`holdings`／`investments`／`users` 追加）に**作り直す**前提。
- 各 DDL は MySQL 8.0 / InnoDB / utf8mb4（`ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci`）で発行する。
- 適用順は FK 依存に従う: `users` → `threads` → `posts` / `holdings` / `investments` → `world_state`（単独）→ 初期データ INSERT。

## ER図（テキスト）

```
        ┌──────────┐
        │  users   │
        └────┬─────┘
       (user_id) │ (investor_id)
        ┌────────┴─────────┐
        ▼                  ▼
   ┌──────────┐      ┌──────────────┐
   │ holdings │      │ investments  │
   └────┬─────┘      └──────┬───────┘
 (thread_id) │              │ (thread_id)
        ▼                  ▼
        ┌──────────────────────┐
        │       threads        │
        └──────────┬───────────┘
            (thread_id) │
                   ▼
              ┌──────────┐
              │  posts   │
              └──────────┘

   ┌─────────────┐
   │ world_state │  （単独・常に1行）
   └─────────────┘
```

**リレーション**

| 関係 | 多重度 | キー | 説明 |
|---|---|---|---|
| users → holdings | 1:N | holdings.user_id | ユーザーは複数スレの株を持つ |
| threads → holdings | 1:N | holdings.thread_id | スレは複数株主を持つ（holdings は users×threads の中間表） |
| users → investments | 1:N | investments.investor_id | ユーザーの投資ログ |
| threads → investments | 1:N | investments.thread_id | スレへの投資ログ |
| threads → posts | 1:N | posts.thread_id | スレは複数レスを持つ |
| users → threads | 0..1:N | threads.creator_id | スレ作成者（匿名は NULL、表示用） |
| world_state | 単独 | — | 世界フェーズ。常に1行 |

## 整合性メモ

- **投資は1トランザクション**: 出金（users.money 減算）・配当入金（既存株主の users.money 加算）・HP更新（threads.hp / last_decay_at / status）・株発行（holdings 加算・threads.total_shares 加算）・変異判定（mutation_level / max_hp）・監査ログ（investments INSERT）を**ひとつのトランザクション**で実行し、失敗時はロールバックする（doc20 §2.2 / §2.6）。
- **holdings の PK は (user_id, thread_id)**: 同一ユーザー×同一スレは1行。再投資は `shares` への加算 UPSERT（`INSERT ... ON DUPLICATE KEY UPDATE shares = shares + VALUES(shares)`）で扱う。
- **FK と ON DELETE 方針**:
  - `posts.thread_id` → `threads.id`: `ON DELETE CASCADE`（スレ削除でレスも消す）。
  - `holdings.thread_id` → `threads.id`: `ON DELETE CASCADE`。
  - `holdings.user_id` → `users.id`: `ON DELETE CASCADE`。
  - `investments.investor_id` → `users.id` / `investments.thread_id` → `threads.id`: **監査ログのため `ON DELETE RESTRICT`**（履歴は保全。アーカイブ運用前提）。
  - `threads.creator_id` → `users.id`: 表示用のため `ON DELETE SET NULL`（匿名相当に戻す）。
  - `posts.author_id` → `users.id`: 表示用のため `ON DELETE SET NULL`。
- **dead は復活不可**: 現在HP ≤ 0 で `status='dead'`。一覧は `idx_threads_status_created` / `idx_posts_thread` を使い `status='alive'` のみを抽出する（doc20 §2.1）。
- **遅延更新の整合**: 表示時は HP を計算するのみ（書き込み不要）。HP が変わるイベント（投資等）時のみ「現在HP算出 → `hp` に書き戻し → `last_decay_at=NOW()`」で確定する。`world_state` も `NOW() ≥ next_shift_at` のアクセス時にフェーズ抽選＋ `next_shift_at` 更新（doc20 §2.5）。
