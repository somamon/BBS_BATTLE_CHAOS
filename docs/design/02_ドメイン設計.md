# ドメイン設計

> 本書は [20_リデザイン_朽ちると投資.md](20_リデザイン_朽ちると投資.md) を正典として書き直したドメインモデル。
> 用語・数値・配分・名称はすべて正典に従い、超える仕様は発明しない。
> バランス定数は `App\Config\Game` に集約（[20 §3](20_リデザイン_朽ちると投資.md#3-バランス定数appconfiggame-に集約)）。

掲示板は **2層構造**（[20 §1](20_リデザイン_朽ちると投資.md#1-コンセプト)）。

- **匿名掲示板層**: 誰でも匿名でスレ立て・レス投稿（無料・無報酬）。スレもレスも個別に HP を持ち、時間で朽ちて 0 で消滅。
- **投資家層**: 登録ユーザーがスレッドの「株」を売買し、配当と株評価益で総資産を競う。

## Entity 一覧

### User（登録投資家）

投資家層の主体。email + password で登録し、所持金を持つ。

**プロパティ**（[20 §4 users](20_リデザイン_朽ちると投資.md#users登録投資家)）

| プロパティ | 型 | 説明 | 初期値 |
|---|---|---|---|
| id | Ulid (VO) | 一意識別子 (ULID) | 自動生成 |
| email | string | ログインID（一意） | — |
| name | string | 表示名（ランキング表示用） | — |
| passwordHash | string | `password_hash()` の出力 | — |
| money | Money (VO) | 所持金 | `INITIAL_MONEY` = 500 |
| createdAt | DateTimeImmutable | 作成日時 | — |

**振る舞い**

| メソッド | 説明 |
|---|---|
| canAfford(Money $amount): bool | `money ≥ amount` かを判定（投資可否） |
| withdraw(Money $amount): void | 投資による出金。残高不足なら例外 |
| deposit(Money $amount): void | 配当の入金 |
| totalAssets(array $holdings): Money | `money + Σ株評価額`（ランキング・マイページ用） |

### Thread（スレッド）

匿名掲示板の投稿スレッドであり、投資の対象（株が発行される唯一の存在）。HP を持ち朽ち、累計投資で変異する。

**プロパティ**（[20 §4 threads](20_リデザイン_朽ちると投資.md#threads)）

| プロパティ | 型 | 説明 | 初期値 |
|---|---|---|---|
| id | Ulid (VO) | 一意識別子 (ULID) | 自動生成 |
| creatorId | Ulid (VO) \| null | 立てた登録ユーザー（匿名は null、表示用） | null |
| title | string | タイトル | — |
| hp | Hp (VO) | 確定HP（スナップショット） | `THREAD_INIT_HP` = 300 |
| maxHp | Hp (VO) | HP上限（変異で上昇） | `THREAD_MAX_HP` = 1000 |
| decayPerMin | int | 減衰レート（基礎値、実減衰は×フェーズ倍率） | `THREAD_DECAY_PER_MIN` = 5 |
| mutationLevel | MutationLevel (VO) | 変異段階（0〜3） | 0 |
| totalShares | Shares (VO) | 発行済み総株数（変異判定に使用） | 0 |
| lastDecayAt | DateTimeImmutable | HP確定時刻 | 作成時刻 |
| status | string | `alive` / `dead` | `alive` |
| postCount | int | レス数（表示・勢いの目安） | 0 |
| createdAt | DateTimeImmutable | 作成日時 | — |
| updatedAt | DateTimeImmutable | 更新日時 | — |

**振る舞い**

| メソッド | 説明 |
|---|---|
| currentHp(Phase $phase, DateTimeImmutable $now): Hp | 遅延減衰で現在HPを算出（書き込みなし、[20 §2.1](20_リデザイン_朽ちると投資.md#21-遅延減衰lazy-decay-実装の肝)）。`max(0, stored_hp − decay_per_min × phase_multiplier × 経過分数)` |
| decayAndPersist(Phase $phase, DateTimeImmutable $now): void | 現在HPを算出 → `hp` に書き戻し → `lastDecayAt = now`。HP≤0なら `status='dead'` |
| isAlive(): bool | `status === 'alive'` |
| invest(Money $amount, ...): InvestResult | 投資の配分計算（HP回復50%/配当30%/sink20%）・株発行・変異判定の起点（[20 §2.2](20_リデザイン_朽ちると投資.md#22-投資株式モデル-経済の心臓)） |
| applyHpGain(Money $hpPortion): Money | `hp += hpPortion`（`maxHp` で上限、超過分は sink へ返す） |
| issueShares(Shares $amount): void | `totalShares += amount`（投資者へ amount 株発行） |
| mutateIfEligible(): void | `totalShares` が `MUTATION_TIERS` 閾値を超えたら `mutationLevel++`、`maxHp` 引き上げ（[20 §2.6](20_リデザイン_朽ちると投資.md#26-変異化け株-スレの成長イベント)） |
| dividendBonus(): float | 現 `mutationLevel` の配当ボーナス（`MUTATION_DIV_BONUS`、Lv0は×1.0） |

> レスは投資対象外なので、Thread のみが株・変異・total_shares を持つ。

匿名掲示板のレス。HP を持ち朽ちるが、株・変異・配当には一切関与しない。

| プロパティ | 型 | 説明 | 初期値 |

| authorHash | string | 匿名識別（IPハッシュ） | — |
| authorId | Ulid (VO) \| null | 登録ユーザーなら紐付け（表示用） | null |
| content | string | 本文 | — |
| hp | Hp (VO) | 確定HP | `POST_INIT_HP` = 100 |
| decayPerMin | int | 減衰レート | `POST_DECAY_PER_MIN` = 5 |
| lastDecayAt | DateTimeImmutable | HP確定時刻 | 作成時刻 |
| status | string | `alive` / `dead` | `alive` |
| createdAt | DateTimeImmutable | 作成日時 | — |

**振る舞い**

| メソッド | 説明 |
|---|---|
| currentHp(Phase $phase, DateTimeImmutable $now): Hp | 遅延減衰で現在HPを算出（Thread と同式。世界フェーズ倍率を共有） |
| decayAndPersist(Phase $phase, DateTimeImmutable $now): void | 現在HPを書き戻し、HP≤0で `status='dead'` |

### Investment（投資監査ログ）

1回の投資の配分内訳を残す不変の監査ログ。経済の検証・履歴表示に用いる。

**プロパティ**（[20 §4 investments](20_リデザイン_朽ちると投資.md#investments監査ログ)）

| プロパティ | 型 | 説明 |
|---|---|---|
| id | Ulid (VO) | 一意識別子 (ULID) |
| investorId | Ulid (VO) | 投資者 (FK users) |
| threadId | Ulid (VO) | 対象スレ |
| amount | Money (VO) | 投資総額 |
| toHp | Money (VO) | HPへ充当した額（50%、超過は sink へ） |
| toDividend | Money (VO) | 配当に回した額（30%、既存株主不在なら sink へ合流） |
| toSink | Money (VO) | 消滅額（20% ＋ 余剰分） |
| createdAt | DateTimeImmutable | 発生日時 |

> 配分は常に `amount = toHp + toDividend + toSink` を満たす（監査整合）。Investment は生成後に変更しない（イミュータブル）。

### WorldState（世界フェーズ・常に1行）

世界に1行だけ存在する相場環境（[20 §2.5](20_リデザイン_朽ちると投資.md#25-世界フェーズ相場天候-市場全体のボラティリティ)）。全スレの減衰に倍率を掛ける。

**プロパティ**（[20 §4 world_state](20_リデザイン_朽ちると投資.md#world_state世界フェーズ常に1行)）

| プロパティ | 型 | 説明 | 初期値 |
|---|---|---|---|
| id | int | 固定値 1 (PK) | 1 |
| phase | Phase (VO) | `boom` / `calm` / `storm` / `crash` | `calm` |
| phaseMultiplier | float | 現フェーズの減衰倍率 | 1.0 |
| nextShiftAt | DateTimeImmutable | 次にフェーズ抽選する時刻（遅延遷移） | — |
| updatedAt | DateTimeImmutable | 更新日時 | — |

**振る舞い**

| メソッド | 説明 |
|---|---|
| multiplier(): float | 現フェーズの減衰倍率（遅延減衰計算に渡す） |
| shouldShift(DateTimeImmutable $now): bool | `now ≥ nextShiftAt` か |
| shift(DateTimeImmutable $now): void | フェーズを抽選し直し（`PHASES`）、`phaseMultiplier` を更新、`nextShiftAt = now + rand(PHASE_MIN_SEC, PHASE_MAX_SEC)` |

> 遅延遷移: cron は使わず、任意アクセス時に `shouldShift()` なら `shift()`（[20 §2.5](20_リデザイン_朽ちると投資.md#25-世界フェーズ相場天候-市場全体のボラティリティ)）。フェーズ切替時に全スレを書き戻さず、現フェーズ倍率で近似する。

## 値オブジェクト（Value Object）

| VO名 | 内部型 | バリデーション / 不変条件 |
|---|---|---|
| Ulid | string | ULID 形式（26文字）。id 群の共通表現 |
| Money | int | `≥ 0`。所持金・投資額・配分額。`amount ≥ MIN_INVEST`(=1) を別途検証 |
| Hp | int | `0 ≤ hp ≤ max_hp`。下限0・上限 `max_hp`。0 で dead 判定 |
| Shares | int | `≥ 0`。保有株・総株。比率計算の分母 |
| MutationLevel | int | `0〜3`（[20 §2.6](20_リデザイン_朽ちると投資.md#26-変異化け株-スレの成長イベント)）。段階のみ上昇（不可逆） |
| Phase | enum | `boom` / `calm` / `storm` / `crash`。各々が減衰倍率を持つ（0.7 / 1.0 / 1.8 / 2.5） |

## ドメインルール

### HP（朽ちる）

- 現在HP `= max(0, stored_hp − decay_per_min × phase_multiplier × 経過分数)`。**下限 0**、**上限 `max_hp`**（Thread のみ。Post は `POST_INIT_HP` が事実上の上限）。
- 投資など HP が変わるイベント時のみ「現在HPを算出 → `hp` に書き戻し → `last_decay_at = now`」で確定。表示時は計算のみ（書き込み不要）。
- **現在HP ≤ 0 で `status='dead'`**。dead は一覧から除外され、**復活不可**（再投資・再活性も不可）。

### 投資（[20 §2.2](20_リデザイン_朽ちると投資.md#22-投資株式モデル-経済の心臓)）

- 投資は **alive スレ限定**。dead には投資不可。
- `amount ≥ MIN_INVEST`(=1) かつ `User.money ≥ amount`。満たさなければ拒否。
- 配分: **HP回復 50%** / **配当 30%** / **sink 20%**（`SPLIT_HP` / `SPLIT_DIVIDEND` / `SPLIT_SINK`）。
- HP回復は `max_hp` で上限。**超過分は sink へ**回す。
- 投資者は **`amount` 株**を受け取り、`thread.total_shares += amount`。
- 配当は **新規株を発行する前の既存株主**へ、各自の **持ち分比率**（`my_shares ÷ total_shares`）で分配。投資者本人は対象外。
- **最初の投資者**は既存株主が居ないため、その回の配当(30%)は **sink に合流**。
- **自己投資**: 自分が立てた/既に株主のスレへの投資も可能だが、配当は「新規発行前の既存株主」基準で計算され、**投資者本人（その回の投資分）は配当を受けない**。それ以前から保有していた他株主のみが受け取る。
- 変異スレへの新規投資は配当配分(30%)に `dividendBonus()`(`MUTATION_DIV_BONUS`) を掛けて株主に厚く分配し、その分 sink を減らす。
- 全工程（出金・配当入金・HP更新・株発行・変異判定・ログ）を**1トランザクション**で行い、失敗時はロールバック。

### 株評価額（[20 §2.3](20_リデザイン_朽ちると投資.md#23-株の評価額)）

- 評価額 `= (my_shares ÷ total_shares) × スレの現在HP`。
- HP が朽ちれば評価額・配当も枯れ、**dead で株は紙くず**（評価額 0）。

### 変異（[20 §2.6](20_リデザイン_朽ちると投資.md#26-変異化け株-スレの成長イベント)）

- 判定は**投資トランザクション内**（イベント駆動、遅延ではない）。
- `total_shares` が `MUTATION_TIERS`（Lv1:500 / Lv2:2000 / Lv3:8000）の閾値を超えると `mutation_level++`。
- 各 Lv で `max_hp`（2000 / 4000 / 8000）と配当ボーナス（1.1 / 1.2 / 1.3）が強化される。Lv は上昇のみで戻らない。

### お金（[20 §2.4](20_リデザイン_朽ちると投資.md#24-お金の出入りまとめ)）

- 登録時に `INITIAL_MONEY` = 500。レス投稿・スレ作成は**無料・無報酬**。
- 増えるのは「自分が株主のスレへ新規投資が入ったときの配当」のみ。sink 20% ＋ HP化 50% は経済から退場（デフレ・有限ラウンド、[20 §8](20_リデザイン_朽ちると投資.md#8-経済特性重要な留意点)）。

## 集約と関係

```
User ──(Investment: 監査ログ)── Thread
                                   │
                                   ├─ Holding（持ち分。Thread 集約に紐づく）
                                   └─ Post（レス。投資対象外）
WorldState（世界に1行。全 Thread / Post の減衰へ倍率を供給）
```

- **Thread 集約**: `Thread` を集約ルートとし、その株主台帳である `Holding` 群と、レスである `Post` 群が紐づく。`total_shares` と各 `Holding.shares` の整合（`Σ Holding.shares = total_shares`）は集約内で保たれる。
- **User と Thread の関係**: 直接の関連は持たず、**`Investment`（投資行為の監査ログ）と `Holding`（保有の結果）を通じて結ばれる**。投資という1イベントが、User の出金・既存株主への配当・Thread の HP/株/変異更新・Investment ログ生成をまたいで作用する（[20 §2.2](20_リデザイン_朽ちると投資.md#22-投資株式モデル-経済の心臓)）。
- **WorldState**: どの集約にも属さない単独の世界状態（1行）。`phase_multiplier` を遅延減衰計算へ供給し、全 Thread / Post の朽ちる速度に影響する。

> 投資ロジックの一貫性は単一ユースケース（`InvestService::invest()`）に集約し、トランザクション境界をそこに置く（[20 §7](20_リデザイン_朽ちると投資.md#7-アーキテクチャ方針)）。減衰計算は Model（`Thread::currentHp()` / `decayAndPersist()`）に集約して二重化を防ぐ。
