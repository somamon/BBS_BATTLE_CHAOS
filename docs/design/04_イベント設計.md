# イベント設計

## イベント駆動アーキテクチャ

投稿を起点に連鎖的にイベントが発生する。
ドメインイベント → リスナー → 副作用 の流れで疎結合を実現。

---

## ドメインイベント一覧

### Post 系

| イベント | 発火タイミング | ペイロード |
|---|---|---|
| PostCreated | 投稿が保存された後 | postId, threadId, content, authorHash |
| PostCorrupted | 投稿が汚染された時 | postId, threadId |
| PostRevived | 古い投稿が復活した時 | postId, threadId |

### Thread 系

| イベント | 発火タイミング | ペイロード |
|---|---|---|
| ThreadCreated | スレッドが作成された時 | threadId, title |
| ThreadDamaged | スレッドがダメージを受けた時 | threadId, damage, remainingHp |
| ThreadCollapsed | スレッドが崩壊した時 | threadId |
| ThreadMutated | スレッドが変異完了した時 | threadId, mutationLevel |
| ThreadFrozen | スレッドが凍結された時 | threadId, duration |
| ThreadWeatherChanged | スレッドの天候が変わった時 | threadId, oldWeather, newWeather |

### World 系

| イベント | 発火タイミング | ペイロード |
|---|---|---|
| WorldWeatherChanged | 世界天候が変わった時 | oldWeather, newWeather |
| WorldPhaseChanged | フェーズが遷移した時 | oldPhase, newPhase |
| WorldDangerIncreased | 危険度が上昇した時 | oldLevel, newLevel |
| WorldEventTriggered | 世界イベントが発動した時 | eventType, scope |

### Challenge 系

| イベント | 発火タイミング | ペイロード |
|---|---|---|
| ChallengeRotated | お題が入れ替わった時 | oldChallengeId, newChallengeId, rule |
| ChallengeViolated | お題ルール違反が発生した時 | challengeId, postId, penaltyType |

## ゲームイベント（プレイヤーに見えるイベント）

投稿時の乱数判定で発生する、世界観上のイベント。

### イベント定義

| イベントID | 名前 | 影響範囲 | 効果 |
|---|---|---|---|
| meteor | 隕石落下 | thread | 対象スレHP -30 |
| red_rain | 赤い雨 | world | 世界天候を「赤い雨」に、全スレ汚染+5 |
| blackout | 停電 | world | 全スレ30秒凍結 |
| garbled_text | 文字化け | thread | 対象スレの直近5投稿を文字化け |
| thread_split | スレ分裂 | thread | 対象スレから投稿の半分が新スレに移動 |
| post_revive | 古いレス復活 | thread | ランダムな過去投稿が最新として再表示 |
| corruption_wave | 汚染波 | world | 全スレ汚染度+20 |
| time_reverse | 時間逆行 | thread | 対象スレの投稿順序が一時的に逆転 |
| curse | 呪い | thread | 投稿者の次の投稿が自動で文字化け |
| earthquake | 地震 | world | 全スレHP -10、ランダムで1スレ崩壊 |
| healing_light | 回復の光 | thread | 対象スレHP +30、汚染度-10 |
| void_expansion | 虚無拡大 | world | 世界天候を「虚無」に、異常度+15 |

### イベント発生率テーブル

**基本発生率: 15%（投稿1回あたり）**

| イベント | 基本重み | 条件ブースト |
|---|---|---|
| meteor | 10 | フェーズchaos以上で ×2 |
| red_rain | 8 | 深夜帯で ×1.5 |
| blackout | 5 | 深夜帯で ×2 |
| garbled_text | 15 | 汚染度50以上で ×2 |
| thread_split | 3 | 投稿数50以上で解禁 |
| post_revive | 10 | 投稿数20以上で解禁 |
| corruption_wave | 8 | フェーズunstable以上で解禁 |
| time_reverse | 5 | 異常度50以上で解禁 |
| curse | 10 | 汚染度30以上で解禁 |
| earthquake | 5 | フェーズchaos以上で解禁 |
| healing_light | 12 | 常時（バランス調整枠） |
| void_expansion | 3 | フェーズchaos以上で解禁 |

**発生率計算**

```
1. 基本発生率(15%) × 各種倍率 → イベント発生判定
2. 発生時、重みテーブルから加重ランダムでイベント選択
3. 条件未達のイベントはテーブルから除外
```

## イベントリスナー

ドメインイベントを受け取り、副作用を実行する。

### MeteorListener

```
受信: WorldEventTriggered (type: meteor)
処理:
  1. 対象スレッドに HP -30
  2. スレッドHP が 0 以下なら CollapseThread 実行
  3. 世界の危険度 +3
```

### CorruptionListener

```
受信: PostCreated
処理:
  1. スレッドの汚染度チェック
  2. 汚染度80以上なら、20%の確率で投稿を自動汚染
  3. 汚染度100でスレッド強制変異
```

### NightmareListener

```
受信: PostCreated（深夜帯のみ）
処理:
  1. 深夜帯（0:00〜5:00）判定
  2. 追加イベント抽選（通常とは別枠で10%追加）
  3. 深夜限定イベントの解禁
     - 血月: 世界天候変更、全スレに特殊演出
     - 怪文書: ランダムなスレに意味不明な投稿が自動生成
```

### PhaseTransitionListener

```
受信: WorldPhaseChanged
処理:
  newPhase に応じた処理:
  - unstable: 「世界が不安定になりました」イベントログ追加
  - chaos: 全スレの天候をランダム変更
  - apocalypse: 終末カウントダウン開始（投稿100回で世界リセット）
  - calm: 全スレHP回復、汚染度半減、天候リセット
```

### CollapseChainListener

```
受信: ThreadCollapsed
処理:
  1. 崩壊したスレッドの汚染度を近隣スレッドに伝播
  2. 世界の危険度 +5
  3. 崩壊スレ数が全体の50%を超えたら世界イベント発動
```

## イベントディスパッチャー

**設計方針**

- 同期ディスパッチ（PHPなのでシンプルに）
- リスナーは優先度付きで登録
- 1イベントに複数リスナー登録可能

**登録例**

```
PostCreated
  → CorruptionListener (優先度: 10)
  → NightmareListener (優先度: 20)

ThreadCollapsed
  → CollapseChainListener (優先度: 10)

WorldPhaseChanged
  → PhaseTransitionListener (優先度: 10)
```

## 時間ベースイベント

投稿トリガーではなく、時間経過で発生するイベント。

**実装方式**

ポーリングリクエスト時に時間チェックを行う（cron不要）。

| チェック項目 | 間隔 | 処理 |
|---|---|---|
| お題ローテーション | 30〜60分 | 期限切れのお題を入れ替え |
| 天候自然変化 | 15分 | 確率で世界天候が変化 |
| 汚染自然減少 | 10分 | 全スレの汚染度を -1 |
| 危険度自然減少 | 5分 | 世界の危険度を -1 |
| 深夜帯切替 | リアルタイム | 0:00 / 5:00 で深夜モード ON/OFF |
