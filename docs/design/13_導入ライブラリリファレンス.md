# 13. 導入ライブラリ リファレンス

`composer require` で `vendor/` 配下に導入した外部ライブラリの一覧と使い方をまとめる。

## インストール済みパッケージ一覧

| パッケージ | バージョン | 役割 | 区分 |
|---|---|---|---|
| `nikic/fast-route` | 1.3.0 | URLルーティング | 直接利用 |
| `league/plates` | 3.6.0 | テンプレート（ビュー） | 直接利用 |
| `php-di/php-di` | 7.1.1 | DIコンテナ（依存性注入） | 直接利用 |
| `php-di/invoker` | 2.3.7 | php-di の依存（呼び出し補助） | 間接 |
| `psr/container` | 2.0.2 | php-di の依存（PSR-11 インターフェース） | 間接 |
| `laravel/serializable-closure` | 2.0.13 | php-di の依存（クロージャのシリアライズ） | 間接 |

> 「間接」は php-di が内部で使う依存パッケージ。自分で直接 import することは基本ない。

導入コマンド（再掲・Docker）:

```bash
docker run --rm -v "$PWD":/app -w /app composer:2 require nikic/fast-route league/plates php-di/php-di
```

役割分担のイメージ:

- **fast-route** … どのURL → どのコントローラか（経路）
- **php-di** … そのコントローラと依存をどう組み立てるか（生成・配線）
- **plates** … コントローラが最後にどうHTMLを描画するか（表示）

## 1. nikic/fast-route 1.3.0 — ルーティング

正規表現ベースの高速ルーター。現状の自作 [Router.php](../../src/Presentation/Routing/Router.php) を置き換える役割。`{id}` のようなURLパラメータが使えるのが自作版との大きな差。

### 基本

```php
use FastRoute\RouteCollector;
use function FastRoute\simpleDispatcher;

$dispatcher = simpleDispatcher(function (RouteCollector $r) {
    $r->addRoute('GET',  '/thread',          ['App\Presentation\Controller\ThreadController', 'index']);
    $r->addRoute('GET',  '/thread/{id:\d+}', ['App\Presentation\Controller\ThreadController', 'show']);
    $r->addRoute('POST', '/res',             ['App\Presentation\Controller\ResController', 'store']);
});
```

### ディスパッチ（index.php 側）

```php
$httpMethod = $_SERVER['REQUEST_METHOD'];
$uri        = rawurldecode(parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH));

$routeInfo = $dispatcher->dispatch($httpMethod, $uri);

switch ($routeInfo[0]) {
    case FastRoute\Dispatcher::NOT_FOUND:
        http_response_code(404);
        echo 'Not Found';
        break;
    case FastRoute\Dispatcher::METHOD_NOT_ALLOWED:
        http_response_code(405);
        echo 'Method Not Allowed';
        break;
    case FastRoute\Dispatcher::FOUND:
        [$class, $method] = $routeInfo[1];   // ハンドラ
        $vars             = $routeInfo[2];   // 例: ['id' => '42']
        (new $class())->$method($vars);
        break;
}
```

### ポイント

- `dispatch()` の戻り値は `[ステータス, ハンドラ, パラメータ配列]`。
- プレースホルダは `{name}`、型制約付きは `{id:\d+}`（数値のみ）など正規表現を付与できる。
- ルート定義をキャッシュしたい場合は `simpleDispatcher` の代わりに `FastRoute\cachedDispatcher` を使う（本番での高速化）。

## 2. league/plates 3.6.0 — テンプレート（ビュー）

素のPHPをそのままテンプレートに使う軽量エンジン。コンパイル不要・独自構文なし。現状コントローラで HTML を直接 `echo` している箇所（[PlaygroundController.php](../../src/Presentation/Controller/PlaygroundController.php) 等）をビューファイルに分離するのに使う。

### セットアップ

```php
use League\Plates\Engine;

// テンプレート置き場のディレクトリを指定
$templates = new Engine('/var/www/html/src/Presentation/View');
```

### コントローラからの描画

```php
echo $templates->render('thread/index', [
    'title'   => 'スレッド一覧',
    'threads' => $threads,   // ユースケースから受け取ったデータ
]);
```

### テンプレート例 `src/Presentation/View/thread/index.php`

```php
<?php $this->layout('layout', ['title' => $title]) ?>

<h1><?= $this->e($title) ?></h1>
<ul>
  <?php foreach ($threads as $t): ?>
    <li><?= $this->e($t['name']) ?></li>
  <?php endforeach ?>
</ul>
```

### レイアウト例 `src/Presentation/View/layout.php`

```php
<!DOCTYPE html>
<html lang="ja">
<head><meta charset="UTF-8"><title><?= $this->e($title) ?></title></head>
<body>
  <?= $this->section('content') ?>
</body>
</html>
```

### ポイント

- `$this->e($value)` … **HTMLエスケープ（XSS対策）**。出力は基本これを通す。
- `$this->layout('layout', [...])` … 共通レイアウトを指定。
- `$this->section('content')` … 子テンプレートの中身をレイアウトに差し込む。
- `$this->insert('partials/header')` … 部分テンプレートの読み込み。

## 3. php-di/php-di 7.1.1 — DIコンテナ（依存性注入）

コントローラやユースケースが必要とする依存（リポジトリ等）を自動で組み立てて注入する。「Controller → UseCase → Repository」をつなぐ配線役。**v7 系は PHP 8 以上が必須**（本プロジェクトの php コンテナは 8.3 なのでOK）。

### 最小（autowiring：型ヒントから自動解決）

```php
use DI\ContainerBuilder;

$builder   = new ContainerBuilder();
$container = $builder->build();

// new せずにコンテナ経由で生成 → コンストラクタの依存が自動注入される
$controller = $container->get(App\Presentation\Controller\ThreadController::class);
```

### インターフェース → 実装のひも付け（定義ファイル `config/di.php`）

```php
use Psr\Container\ContainerInterface;
use function DI\autowire;

return [
    // ThreadRepositoryInterface を要求されたら MySQLThreadRepository を渡す
    App\Domain\Repository\ThreadRepositoryInterface::class
        => autowire(App\Infrastructure\Repository\MySQLThreadRepository::class),

    // ファクトリで組み立てたいものはクロージャで（PDO接続など）
    PDO::class => function (ContainerInterface $c) {
        return new PDO(
            'mysql:host=db;dbname=bbs;charset=utf8mb4',
            'bbs',
            'bbs',
            [PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION]
        );
    },
];
```

### 定義ファイルの読み込み

```php
$builder = new DI\ContainerBuilder();
$builder->addDefinitions(__DIR__ . '/../config/di.php');
$container = $builder->build();
```

### ポイント

- コンストラクタの型ヒントを辿って依存を芋づる式に自動生成する（autowiring）。
- インターフェースを実装にマップするには `DI\autowire()` を使う。
- 動的に組み立てたいものは無名関数（ファクトリ）で定義。
- 本番では `$builder->enableCompilation(__DIR__ . '/cache')` でコンパイルして高速化できる。

## 4. 3つを組み合わせた index.php の全体像

```php
require_once __DIR__ . '/../vendor/autoload.php';

// php-di: コンテナ構築
$container  = (new DI\ContainerBuilder())
    ->addDefinitions(__DIR__ . '/../config/di.php')
    ->build();

// fast-route: ルート定義をロードしてディスパッチャ生成
$dispatcher = FastRoute\simpleDispatcher(require __DIR__ . '/../config/routes.php');

$info = $dispatcher->dispatch(
    $_SERVER['REQUEST_METHOD'],
    rawurldecode(parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH))
);

if ($info[0] === FastRoute\Dispatcher::FOUND) {
    [$class, $method] = $info[1];
    $controller = $container->get($class);   // php-di が依存を注入
    $controller->$method($info[2]);          // fast-route のURLパラメータを渡す
    // → コントローラ内部で plates の render() を呼んでHTMLを返す
} else {
    http_response_code(404);
    echo 'Not Found';
}
```

現状の自作実装からの置き換え対応:

| 現状 | 置き換え先 |
|---|---|
| 自作 [Router.php](../../src/Presentation/Routing/Router.php) の `dispatch()` | fast-route |
| `(new $controller())->$action()`（直接 new） | php-di の `$container->get()` |
| コントローラ内で HTML を直接 `echo` | plates の `render()` + ビューファイル |
