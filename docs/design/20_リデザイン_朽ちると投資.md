# リデザイン: 朽ちる掲示板と投資経済（株式モデル）

> このドキュメントが新しいコア設計の正。00〜10 の各設計書は本書に従って書き直す。
> 旧「異常掲示板」のカオス要素は **相場メカニクス** として再解釈し、
> **世界フェーズ(相場天候)** と **変異(化け株)** をコアに採用（§2.5 / §2.6）。
> 汚染・お題・連鎖イベントは将来拡張（§10）。残課題は §9。

## 1. コンセプト

**2層構造**で成り立つ。

```
┌─ 投資家層（登録ユーザーのみ）────────────────┐
│  お金(初期500)で「スレッドの株」を売買し、配当と    │
│  株評価益で総資産を競う。                          │
└──────────────────▲─────────────────┘
                       │ 賑わいが「将来性シグナル」になる
┌─ 匿名掲示板層（誰でも）──┴─────────────────┐
│  匿名でスレ立て・レス投稿（お金不要・報酬なし）。     │
│  スレもレスも時間で朽ちて消える。                   │
└──────────────────────────────────┘
```

3つの概念:

1. **朽ちる** — スレもレスも個別に HP を持ち、時間で減り、0 で消滅。
2. **投資** — 登録ユーザーがスレッドの株を買う。お金の50%がそのスレのHP回復に充たり延命される。
3. **お金** — 登録時に 500。レス報酬は無し。**自分が株主のスレに新規投資が入ると配当**で増える。

**ゴール（競争）**: 総資産（所持金 ＋ 保有株の評価額）ランキングで上位を狙う。

## 2. 主要メカニクス

### 2.1 遅延減衰（Lazy Decay）— 実装の肝

cron で全行を減らさず、**読むたびに経過時間から現在 HP を計算**する。

```
現在HP = max(0, stored_hp − decay_per_min × phase_multiplier × 経過分数)
経過分数 = (now − last_decay_at) ÷ 60秒
phase_multiplier = 現在の世界フェーズの減衰倍率（§2.5。平常は 1.0）
```

- `stored_hp` / `last_decay_at` は「HP を確定させた時刻のスナップショット」。
- 投資など HP が変わるイベント時に「現在HPを算出 → stored_hp に書き戻し → last_decay_at=now」で確定。
- 表示時は計算だけ（書き込み不要）。
- 現在HP ≤ 0 で `status='dead'`。dead は一覧から除外、**復活不可**。

### 2.2 投資（株式モデル）— 経済の心臓

登録ユーザーがスレッドに `amount`（≥1、所持金以下）を投資すると:

| 配分 | 割合 | 行き先 |
|---|---|---|
| HP回復 | **50%** | 対象スレの `hp += 0.5 × amount`（`max_hp` で上限、超過分は sink へ） |
| 配当 | **30%** | **既存株主**へ持ち分比率で分配（投資者本人は対象外） |
| 消滅(sink) | **20%** | 経済から消える（インフレ抑制） |

- 投資者は **`amount` 株**を受け取る（`holdings` に加算、`thread.total_shares += amount`）。
- 配当は「新規株を発行する**前**の既存株主」に配る。**最初の投資者は既存株主が居ない**ので、その回の配当(30%)は sink に合流。
- dead のスレには投資不可。
- 全工程を**1トランザクション**（出金・配当入金・HP更新・株発行・ログ）で行い、失敗時ロールバック。

**穴①②がこれで解ける:**
- 他人に投資する動機 = 自分が株主になり、**後続投資の配当(30%)**を得る（早期参入が得）。
- 「貯め込み最強」問題の解消 = 投資しないと配当ストリームに乗れない。

### 2.3 株の評価額

```
保有株の評価額 = (my_shares ÷ thread.total_shares) × スレの現在HP
```

HP は時間で溶けるので、**賑わいが止まれば評価額も配当も枯れ、dead で株は紙くず**。
「伸びるうちに乗って、朽ちる前に配当を回収」する投機ゲームになる。

### 2.4 お金の出入りまとめ

| 増える | 減る |
|---|---|
| 登録時 +500 | スレへの投資 −amount |
| 自分が株主のスレに新規投資 → 配当(投資額×30%×持ち分比率) | （sink 20% と HP化 50% で経済から退場） |

レス投稿・スレ作成は**無料・無報酬**（匿名掲示板行為）。

### 2.5 世界フェーズ（相場天候）— 市場全体のボラティリティ

世界に1行だけの `world_state` を持ち、**全スレの減衰に倍率(`phase_multiplier`)を掛ける**「相場環境」を表現する。

| フェーズ | 倍率 | 体感 |
|---|---|---|
| `boom`（活況） | ×0.7 | 朽ちにくい。強気相場 |
| `calm`（平穏） | ×1.0 | 通常 |
| `storm`（嵐） | ×1.8 | 急速に朽ちる。延命コスト増 |
| `crash`（暴落） | ×2.5 | 投げ売り誘発。投資妙味も最大 |

- **遅延遷移**: cron は使わない。アクセス時に `now ≥ next_shift_at` ならフェーズを抽選し直し、`next_shift_at` を更新（1フェーズ＝数分〜十数分）。
- **近似**: 遅延減衰の計算は「現在フェーズの倍率 × 経過分」で近似する（フェーズ切替時に全スレを書き戻さない）。フェーズは頻繁には変わらない前提でMVPは許容。厳密化は将来。
- フェーズはトップ／一覧ヘッダーに常時表示（"いまの相場"）。嵐・暴落は投資家の行動を一気に動かす。

### 2.6 変異（化け株）— スレの成長イベント

スレへの累計投資（＝`total_shares`）が閾値を超えると **変異** し、`mutation_level` が上がって特性が強化される。

| Lv | 必要 total_shares | max_hp | 配当ボーナス | 演出 |
|---|---|---|---|---|
| 0 | — | 1000 | ×1.0 | 通常 |
| 1 | 500 | 2000 | ×1.1 | 「変異種」バッジ |
| 2 | 2000 | 4000 | ×1.2 | タイトル発光 |
| 3 | 8000 | 8000 | ×1.3 | 「特異点」化 |

- 変異判定は**投資のトランザクション内**で行う（遅延でなくイベント駆動）。閾値到達で `mutation_level++`、`max_hp` 引き上げ。
- 配当ボーナス: 変異スレへの新規投資は配当配分(30%)に `配当ボーナス` を掛けて株主に厚く分配（その分 sink を減らす）。
- 投資家は「変異目前のスレ」を先読みして仕込む → 化け株を当てる興奮。max_hp 上昇で延命の上限も伸び、長く戦える優良株になる。

## 5. ユースケース

| UC | 認証 | 処理 | 失敗条件 |
|---|---|---|---|
| ユーザー登録 | 不要→登録 | email + name + password 受取 → `password_hash()` で保存 → users 作成（money=500）→ セッションに紐付け | メール重複 / 形式不正 |
| ログイン | 不要 | email で引き → `password_verify()` → セッションに user_id を保存 | メール不在 / パス不一致 |
| スレ作成 | 匿名可 | thread 作成（HP満タン, total_shares=0）。登録者なら creator_id 記録 | — |
| レス投稿 | 匿名可 | スレ alive 確認 → post 作成 → post_count++ | スレ dead |
| 投資 | **要登録** | 残高・alive 確認 → §2.2 を1トランザクションで実行 → §2.6 変異判定 | 未登録 / 残高不足 / dead / amount<MIN |
| 相場遷移 | — | 任意アクセス時に `now ≥ next_shift_at` ならフェーズ抽選（§2.5、遅延・cron不要） | — |
| 一覧/詳細 | 匿名可 | 現在フェーズ倍率で現在HPを遅延計算して表示（dead 除外） | — |
| ランキング | 匿名可 | 全ユーザーの `money + Σ株評価額` を集計し降順 | — |
| マイページ | 要登録 | 所持金・保有株・各評価額・含み損益 | 未登録 |

## 7. アーキテクチャ方針

簡易MVC（`App\Model` + `Presentation/Controller|View`）を踏襲。
- 減衰計算は Model に集約（`Thread::currentHp()` / `decayAndPersist()`）し二重化を防ぐ。
- 投資ロジックは1つのユースケース（`InvestService::invest()`）にまとめ、トランザクション境界をそこに置く。
- ゲーム定数は `App\Config\Game` に一元化。
- ログインは email + password（`password_hash()` / `password_verify()`）。セッションに user_id を保持。

## 8. 経済特性（重要な留意点）

レス報酬を 0、配当は再分配（新規お金を生まない）にしたため、**お金は「初期500×登録人数」が天井で、投資のたびに 70%（HP化50%＋sink20%）が循環マネーから退場する＝デフレ**。

➡ **このゲームは構造上「有限ラウンド」**（採用確定／蛇口は足さない）。お金が枯れると新規投資が止まり、全スレが朽ちて終局へ向かう。

**終局条件（どちらか）**:
- 全スレッドが `dead`、または
- 全登録ユーザーの所持金合計 < `MIN_INVEST`（＝誰も投資できない）

**終局処理**: ランキングを確定し「リザルト画面」で勝者（総資産トップ）を表示。以降は新ラウンド開始（全データをアーカイブ→初期化）まで閲覧のみ。
※ 終局判定は投資/閲覧のたびに軽くチェックする遅延方式でよい（cron 不要）。

## 9. 残課題

| # | 項目 | 状態 |
|---|---|---|
| 1 | 有限ラウンド or 無限 | ✅ **有限ラウンド**で確定（§8、蛇口なし） |

| 3 | 数値バランス（減衰・初期HP・配分%） | 🔧 実プレイ後に調整。`App\Config\Game` で集中管理 |
| 4 | スレ作成のスパム対策 | 🔧 実装後に判断（クールダウン等）。MVP は無対策 |

→ 残るは実装時チューニング項目のみ。**コア設計は確定**。次は実装（スキーマ刷新 → Model/UseCase → Controller/View）。

## 10. 将来拡張（旧カオス要素 → 相場メカニクス）

コアを汚さず「相場イベント層」として段階的に追加できる。今回はコア外。

| 拡張 | 旧要素 | 投資ゲームでの意味 |
|---|---|---|
| **汚染（ジャンク化）** | corruption | dead 接近や特定条件で汚染が進み、減衰加速＆配当率低下。優良株/不良株の選別軸 |
| **お題（テーマ株）** | challenges | 定期的にお題が出て、該当スレは期間中、減衰緩和や配当ボーナス。物色セクター |
| **連鎖イベント（市場サプライズ）** | events | 投稿/投資を起点に確率で暴落・高騰・災害が発生。イベントログ＝市場ニュース |

> 採用済みのコア相乗要素は **世界フェーズ(§2.5)** と **変異(§2.6)** の2つ。
