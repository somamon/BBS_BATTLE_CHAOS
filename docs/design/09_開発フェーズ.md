# 開発フェーズ

## 全体方針

- Phase 1 で最小限動くものを作る（投稿→スレ状態変化の体験）
- Phase 2 でゲーム性を追加（イベント、お題、汚染）
- Phase 3 で演出を強化（UI侵食、深夜イベント）
- 各Phase内はステップ単位で進め、ステップごとに動作確認

---

## Phase 1: 基盤（掲示板＋世界状態）

最小限の「動く掲示板」を作る。

### Step 1-1: プロジェクト初期化

| タスク | 詳細 |
|---|---|
| Composer 初期化 | PSR-4 オートロード設定 |
| ディレクトリ作成 | src/ 配下の全ディレクトリ |
| フロントコントローラー | public/index.php |
| ルーター | シンプルなURL→Controller振り分け |

**完了条件**: `php -S localhost:8080 -t public` で「Hello World」が表示される

### Step 1-2: Domain層（Entity / VO / Enum）

| タスク | 詳細 |
|---|---|
| Thread エンティティ | プロパティ、takeDamage、heal 等 |
| Post エンティティ | プロパティ、基本メソッド |
| WorldState エンティティ | プロパティ、increaseDanger 等 |
| ValueObject 群 | ThreadId, HitPoint, Corruption 等 |
| Enum 群 | ThreadStatus, WorldPhase, Weather 等 |
| Repository インターフェース | 全5つ |

**完了条件**: Domain層のクラスが全て定義され、単体で矛盾なく動作する

### Step 1-3: Infrastructure層（DB＋リポジトリ実装）

| タスク | 詳細 |
|---|---|
| SQLiteConnection | PDO接続、WALモード設定 |
| マイグレーション | 全テーブル作成スクリプト |
| SQLiteThreadRepository | CRUD実装 |
| SQLitePostRepository | CRUD実装 |
| SQLiteWorldStateRepository | getCurrent, save 実装 |
| 初期データ投入 | world_states の初期レコード |

**完了条件**: マイグレーション実行後、DB操作が正常に動作する

### Step 1-4: UseCase層（基本機能）

| タスク | 詳細 |
|---|---|
| CreateThread | スレッド作成 |
| GetThread | スレッド詳細取得 |
| ListThreads | スレッド一覧取得 |
| CreatePost（簡易版） | 投稿保存＋スレHP微減のみ |
| GetWorldState | 世界状態取得 |

**完了条件**: UseCaseを手動実行してDB操作ができる

### Step 1-5: Presentation層（画面表示）

| タスク | 詳細 |
|---|---|
| レイアウトテンプレート | layout.php（共通HTML） |
| トップページ | スレ一覧、世界状態表示 |
| スレ詳細ページ | 投稿一覧、投稿フォーム |
| スレ作成ページ | フォーム |
| CSS基本スタイル | ダークテーマ、HPバー、汚染ゲージ |
| HTMX導入 | CDN読み込み、ポーリング設定 |
| CSRF対策 | トークン生成・検証 |

**完了条件**: ブラウザでスレ作成→投稿→HP減少が確認できる

## Phase 2: ゲーム性

投稿がイベントを引き起こし、世界が変化する。

### Step 2-1: イベント基盤

| タスク | 詳細 |
|---|---|
| EventDispatcher | イベント発火・リスナー登録 |
| ドメインイベント定義 | PostCreated, ThreadCollapsed 等 |
| GameEvent エンティティ | イベントログ用 |
| SQLiteEventRepository | イベント保存・取得 |
| イベントログ表示 | 画面上にイベント履歴を表示 |

**完了条件**: 投稿時にドメインイベントが発火し、リスナーが動作する

### Step 2-2: イベント判定

| タスク | 詳細 |
|---|---|
| EventJudgeService | 乱数イベント判定ロジック |
| RandomGenerator | 乱数生成インターフェース＋実装 |
| イベント定義テーブル | 各イベントの効果・重み・条件 |
| CreatePost 拡張 | イベント判定を統合 |

**完了条件**: 投稿するとランダムにイベントが発生し、スレ/世界の状態が変化する

### Step 2-3: イベントリスナー実装

| タスク | 詳細 |
|---|---|
| MeteorListener | 隕石ダメージ処理 |
| CorruptionListener | 汚染連鎖処理 |
| CollapseChainListener | 崩壊連鎖処理 |
| PhaseTransitionListener | フェーズ遷移処理 |

**完了条件**: イベント発生→連鎖反応が自動的に発生する

### Step 2-4: お題システム

| タスク | 詳細 |
|---|---|
| Challenge エンティティ | お題定義 |
| SQLiteChallengeRepository | お題保存・取得 |
| ChallengeJudgeService | ルール適合判定 |
| RotateChallenge | お題ローテーション |
| CreatePost 拡張 | お題違反判定を統合 |
| お題表示UI | 現在のお題＋残り時間 |

**完了条件**: お題が自動ローテーションし、違反投稿にペナルティが発生する

### Step 2-5: 世界イベント

| タスク | 詳細 |
|---|---|
| TriggerWorldEvent | 世界全体に影響するイベント |
| ChangeWeather | 天候変更ロジック |
| UpdatePhase | フェーズ自動遷移 |
| 時間ベースイベント | ポーリング時の時間チェック |

**完了条件**: 世界状態が動的に変化し、全スレに影響を与える

## Phase 3: 演出

UIが世界状態に連動して変化する。

### Step 3-1: CSS演出

| タスク | 詳細 |
|---|---|
| 天候テーマCSS | 天候別の色調・エフェクト |
| 汚染演出 | ノイズ、グリッチ、文字崩壊 |
| フェーズ演出 | UI歪み、揺れ、亀裂 |
| HPバー演出 | 低HP時の点滅、色変化 |

**完了条件**: 天候・汚染度でUIの見た目が変化する

### Step 3-2: イベント演出

| タスク | 詳細 |
|---|---|
| 隕石アニメーション | 画面上から落下する光 |
| 停電演出 | 画面ブラックアウト |
| 地震演出 | 画面振動 |
| 回復演出 | 光のフラッシュ |
| イベント通知UI | フラッシュ表示→自動消去 |

**完了条件**: イベント発生時に視覚的フィードバックがある

### Step 3-3: 深夜イベント

| タスク | 詳細 |
|---|---|
| NightmareListener | 深夜帯限定イベント |
| 深夜UI演出 | 暗転、点滅、怪文書 |
| 深夜限定イベント定義 | 血月、怪文書表示等 |

**完了条件**: 0:00〜5:00 に特殊な現象が発生する

### Step 3-4: Alpine.js 演出制御

| タスク | 詳細 |
|---|---|
| 状態管理 | 天候・フェーズに応じたクラス切替 |
| お題タイマー | カウントダウン表示 |
| イベントフラッシュ | 表示→消去のアニメーション管理 |

**完了条件**: Alpine.js でインタラクティブな演出が動作する

## Phase 間の依存

```
Phase 1（基盤）
  │
  ├── Step 1-1 → 1-2 → 1-3 → 1-4 → 1-5（順序依存）
  │
  ↓
Phase 2（ゲーム性）
  │
  ├── Step 2-1（イベント基盤）
  │     ↓
  ├── Step 2-2（イベント判定）─→ Step 2-3（リスナー）
  │                                    ↓
  ├── Step 2-4（お題）          Step 2-5（世界イベント）
  │     ↑ 2-4 と 2-5 は並行開発可能
  │
  ↓
Phase 3（演出）
  │
  ├── Step 3-1（CSS演出）── 並行 ── Step 3-3（深夜）
  │     ↓
  ├── Step 3-2（イベント演出）
  │     ↓
  └── Step 3-4（Alpine.js）
```

## 並行開発の可能性

Phase 2 以降、複数人で並行開発可能な単位:

| 担当A | 担当B | 条件 |
|---|---|---|
| Step 2-2〜2-3（イベント） | Step 2-4（お題） | イベント基盤(2-1)完了後 |
| Step 3-1（CSS演出） | Step 3-3（深夜イベント） | Phase 2 完了後 |
| Step 3-2（イベント演出） | Step 3-4（Alpine.js） | Step 3-1 完了後 |
