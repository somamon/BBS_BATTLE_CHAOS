## ✅ 解消済みブロッカー（B1〜B4）

実機（docker）で動作確認済み。ユニットテスト 81 件グリーン。

### B1. 確認メールの実送信 ✅
- SMTP 実装 [SmtpMailer](../src/Infrastructure/Mail/SmtpMailer.php)（none/STARTTLS/SSL・AUTH LOGIN・UTF-8 MIME、依存ライブラリなし）を追加。
- [Container](../src/Infrastructure/Container.php) が `MAIL_DRIVER` で `smtp`/`log` を切替。開発は **Mailpit**（docker-compose に追加、UI: `MAILPIT_UI_PORT`）。
- 送信失敗はログに留めて UX を止めない（[RegisterUser](../src/Application/UseCase/Auth/RegisterUser.php)/[ResendVerification](../src/Application/UseCase/Auth/ResendVerification.php)）。
- 検証: HTTPS で登録→Mailpit受信→本文の確認リンク→`/verify`→ログイン成立を確認。

### B2. 本番設定ガード ✅
- [Environment::assertProductionReady()](../src/Config/Environment.php) が本番で「脆弱な DB_PASSWORD / 非https の APP_URL / `MAIL_DRIVER≠smtp`」を検出して**起動拒否**（[index.php](../public/index.php) で fail-fast、詳細は画面に出さずログのみ）。
- `.env.example` を本番向け雛形に整備（弱い既定値は起動拒否されると明記）。`.env` は .gitignore 済み。
- 検証: 本番×脆弱設定→BLOCKED、本番×強固設定→BOOT-OK。

### B3. HTTPS ✅
- [nginx default.conf](../docker/nginx/default.conf): 80→443 恒久リダイレクト、443 TLS終端、`HSTS`、PHPへ `HTTPS on` 伝達（Secure Cookie 有効化）。
- 開発は自己署名証明書（`docker/nginx/certs/`、.gitignore）。本番は実証明書をマウント。compose に 443 ポートを追加。
- 検証: `https://…/threads` 200、`http://…` 301、HSTS ヘッダ確認。

### B4. マイグレーションランナー ✅
- [bin/migrate.php](../bin/migrate.php): `schema_migrations` 台帳で未適用SQLのみ順次適用（初回・更新の両対応・冪等）。
- compose から `docker-entrypoint-initdb.d`（初回のみ方式）を廃止し、**一回限りの `migrate` サービス**＋DB healthcheck＋`depends_on` 条件で起動時に自動適用。
- 検証: 初回10件適用→再実行で0件、台帳とボットseed反映を確認。

> 開発時アクセス: **https://localhost:18443**（自己署名のため警告は承認）。メール確認は Mailpit UI **http://localhost:18025**。

## ✅ 解消済み 🟠高（H1〜H6）

実機（docker）・結合テストで動作確認済み。ユニット 81 件＋結合 1 件グリーン。

### H1. N+1 クエリ解消 ✅
- [RankingQuery](../src/Application/UseCase/Ranking/RankingQuery.php) は `holdings->all()` ＋ `posts->findByIds()`（IN一括）で**クエリ数を一定化**。[MyPageQuery](../src/Application/UseCase/User/MyPageQuery.php) も `findByIds` で一括取得。
- 新メソッド: [HoldingRepository::all()](../src/Domain/Repository/HoldingRepository.php) / [PostRepository::findByIds()](../src/Domain/Repository/PostRepository.php)。
- （EndgameStatus は取得済みコレクションをメモリ走査するだけで N+1 ではなかった。）

### H2. シミュレーションのスロットル ✅
- [BotSimStateRepository::tryClaim()](../src/Domain/Repository/BotSimStateRepository.php) による**原子的占有（楽観的ロック）**で、同時アクセスの二重実行を防ぎ、最短 `BOT_TICK_SECONDS`(30s) 間隔に抑制。検証: 30秒以内の連続GETで投資が増えない。

### H3. ボット資金補充 ✅
- [MarketSimulator](../src/Application/Service/MarketSimulator.php) が資金切れボットを `BOT_REFILL_TO` まで補充（中央銀行的な流動性供給）＝長時間ソロでも相場が止まらない。

### H4. 入力長検証 ✅
- [CreateThread](../src/Application/UseCase/Thread/CreateThread.php)（タイトル≤`THREAD_TITLE_MAX`=255）・[PostReply](../src/Application/UseCase/Thread/PostReply.php)（本文≤`POST_CONTENT_MAX`=2000）。検証: 300字タイトルで **422**（従来は500）。

### H5. 結合テスト＋CI ✅
- [DatabaseFlowTest](../tests/Integration/DatabaseFlowTest.php): 実MySQLで「登録→確認→ログイン→スレ作成→レス→投資→ランキング」を通し検証（`RUN_DB_TESTS=1` でのみ実行）。
- [.github/workflows/ci.yml](../.github/workflows/ci.yml): push/PR で composer 検証・lint・migrate・phpunit（unit+integration、MySQLサービス付き）を自動実行。

### H6. 期限切れ行の掃除 ✅
- [RateLimiter::purgeExpired()](../src/Application/Port/RateLimiter.php) / [EmailVerificationRepository::purgeExpired()](../src/Domain/Repository/EmailVerificationRepository.php) を追加し、占有された tick のたびに実行（cron不要の遅延メンテ）。検証: 失効行が掃除される。

## 🟡 中

- **M1. パスワードリセットが無い**: パスワード忘れの救済手段なし＝実質ロックアウト。リセット用トークン（確認メール基盤を流用）を追加。
- **M2. 終局後のリセットが無い**: [EndgameStatus](../src/Application/UseCase/Endgame/EndgameStatus.php) は終局判定のみで、ラウンド再開/初期化の仕組みが無い。永続運用ならラウンド制が要る。
- **M3. ゲームバランス未調整**: 株価係数・配分・減衰・レベル閾値・ボット挙動は実プレイ前提の暫定値（doc21 §10）。クローズドβでの計測・調整が必要。
- **M4. 観測性が薄い**: ログは error_log のみ。メトリクス/アラート/相関IDなし。障害時の追跡が困難。
- **M5. 法務・コンプライアンス未整備**: 利用規約/プライバシーポリシー無し。「お金を賭ける」表現＋ランキングは、仮想通貨でも地域やアプリ配信規約次第で**ギャンブル的**と見なされ得る。年齢確認/表現の整理を検討。

## ⚪ 低

- **L1. セッションがファイル保存**: 単一ホスト前提。水平スケール時は共有ストア（Redis等）へ。
- **L2. ログイン時に `password_needs_rehash` 未対応**: 将来コスト変更時に旧ハッシュが残る。
- **L3. CSP が `style-src 'unsafe-inline'`**: インラインstyle依存。将来 nonce 化の余地。
- **L4. ライセンス/メタ未設定**: composer.json に license 無し等。

## すでに対応済み（リリースに向けた強み）

- 認証/登録のセキュリティ: 入力検証(VO)・bcrypt・**セッション固定対策**・**タイミング安全ログイン**・**メールアドレス列挙対策（登録/ログイン/再送で一貫）**・CSRF(hash_equals)・**レート制限**・確認トークンのハッシュ保存/使い切り/失効・確認メール再送。
- 出力エスケープで XSS 対策、全クエリ prepared statement で SQLi 対策。
- 全レスポンスに**セキュリティヘッダ**（X-Frame-Options/CSP/nosniff/Referrer-Policy）。
- 開発用 playground（eval）は**削除済み**。
- ソロ/少人数対応: **NPC投資家（ランキングのライバル）**（人間50人以下で稼働）と**人口スケール減衰**（過疎時は朽ち遅い）。
- クリーンアーキテクチャ（Domain/Application/Infrastructure/Presentation）維持、ユニットテスト 78 件グリーン。
